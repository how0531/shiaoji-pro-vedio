# Shioaji Pro 教學影片專案 — 開發日誌

> 2026-07-18 ～ 2026-07-20（進行中）。本日誌記錄從「clone 一個開源交易終端」到「一整套 2K 實錄教學影片系列＋自動化製作管線」的完整工程過程，包含所有踩過的坑與根治方法。

## 0. 專案目標

1. 深度理解 [Sinotrade/shioaji-pro-app](https://github.com/Sinotrade/shioaji-pro-app)（React 19 + TS + Vite 開源交易終端）
2. 產出交易者導向的介紹文與功能操作手冊
3. 製作**全程實際操作錄影**的教學影片系列（鐵律：畫面必須是真的在操作，不做假畫面）
4. 每支影片經獨立檢核：**旁白說 A、畫面就演 A**，逐段對齊
5. 全程模擬環境、帳號遮蔽、無金鑰入鏡

## 2. 影片管線演進

- **v1（EP1–5，已封存 `_archive/`）**：粗剪概念版，旁白與畫面對不準 → 淘汰。
- **v2（T 系列 1080p）**：`topics_spec.py` 建立「旁白↔動作契約」——每段 = {id, action, narration}，Playwright 逐段實錄、edge-tts 逐句配音、逐段對時組裝。
- **v3（現行，全 2K）**：`record-topics.py`（2560×1440 原生、字級 1.3、虛擬游標＋點擊漣漪、金框標註、`＋新增面板` 精確選取、mark/cut_end 裁切點）→ `assemble3.py`（每段旁白配音、畫面 retime 至旁白長度、卡片靜態+fade、字幕燒錄、片頭片尾淡入淡出、BGM 混音）→ `generate-cards.py`（永豐配色 2K 字卡：金色內框＋邊緣暈影）。

## 3. 導演循環與獨立檢核（方法論核心）

**不信自檢。**每支影片完成後由獨立 QC agent 重新抽格（fps=1/2）、逐格閱讀「燒錄字幕 vs 畫面實況」，比對是否「說 A 演 A」，同時掃 PII 與錯誤畫面。收斂實績：

- T 系列第一輪抓 4 支錯配 → 第三輪全 PASS（共重錄 5 支）
- 進6/7/8 第一輪：**三集全中「非交易日送單 → 紅色 500 SolClient 錯誤畫面」**＋一處旁白指涉不存在的卡片 → 重啟模擬 server 換 session＋改「不賭實際掛單」的誠實做法（armed 不送出、條件式旁白）→ 第二輪全 PASS
- 進12 第一輪：QC 抓到「旁白描述了沒發生的操作」（排行榜分頁沒點到、複選沒展開、移除沒發生、金框標到空白）→ 根因是 Playwright 選擇器 scope 錯誤靜默跳過 → 修選擇器＋narration 對齊重錄

**檢核的盲區與補強**：QC 只「看」字幕不「聽」聲音，導致 TTS 跳針 bug 漏網（見 §5.3）。補強：`qc-tools.py` 給檢核裝上耳朵（Whisper 轉寫 vs 預期旁白比對）與尺（逐格像素差抖動偵測）。

## 4. 非交易日誠實原則

盤後/假日沒有即時成交價、掛單會被秒退或噴錯。原則：**畫面照實呈現、旁白用條件句**（「盤中的時候…」），絕不假演成交。典型手法：

- 兩段式下單只 arm 到「確認買進」不送出（畫面與「再按一下才送出」旁白天然對齊）
- 委託分頁用觀念式旁白（「送出以後會出現在這裡…」），不指涉特定單
- 圖上停損/停利只切模式看提示條、不點價位（避免「尚未收到即時成交價」紅字）
- web 版沒有的功能（基本面篩選、維持率、希臘值）→ 誠實卡明講「這裡沒有、去哪做」

## 5. 三個重大 bug：根因與根治

### 5.1 非交易日送單 → 持續紅色 500 SolClient 錯誤
模擬 server 跑太久 Solace session 失效，實際送單觸發常駐錯誤畫面污染三集。根治：重啟 server 換新 session＋所有示範改為不實際送單（armed-only）。

### 5.2 字卡抖動（用戶回報）
質感升級時用 ffmpeg `zoompan` 給字卡加慢速推拉。zoompan 每格位移只能取整數像素，慢速縮放理論位移 <0.1px/格 → 實際變成「不動/跳 1px」交錯 = 文字邊緣持續抖動。根治：**文字卡一律靜態＋淡入淡出**（廣播級字卡本來就不 zoom 文字），程式加註防止回歸；`qc-tools.py jitter` 可量化驗證（舊版實測 8.76、靜態版應 <0.8）。

### 5.3 旁白跳針（用戶回報）
TTS 快取按「句序」命名（s000.mp3…），**改旁白後重組會按編號誤用舊句音檔**——嘴唸舊句、字幕是新句，新舊相近時聽起來像跳針。QC 看字幕看不出來，一路漏網。根治：快取改**內容雜湊定址**（文字變→檔名變→必重生成），全量清除舊快取重出 16 支。這類錯誤從結構上不再可能發生。

## 6. 質感升級（用戶：「更有質感一點」）

- 字卡：金色內框＋邊緣暈影＋更深漸層（設計給質感，不靠晃動）
- 字幕：加粗、放大、深陰影、字距；卡片內容上移，**字幕不再壓到卡片文字**
- 片頭片尾：畫面＋聲音淡入淡出
- BGM：Mixkit「Romantic 05」（Underscore/Corporate/Calm，免署名可商用），−20dB 墊底、循環、淡入淡出，`assets/bgm/` 丟檔即用

## 7. 安全與 PII

- 全程模擬環境徽章入鏡、帳號 UI 遮蔽（9A92-••••00 式）

## 11. 工程教訓

1. **自檢不可信**，獨立檢核要能「看」也要能「聽」與「量」。
2. **快取一律內容定址**——位置定址快取在內容會被編輯的系統裡必炸。
3. 慢速 zoompan 不適用於文字；**質感來自設計，不是晃動**。
4. 非交易日限制不是障礙，是「誠實敘事」的設計約束。
5. 選擇器要用已驗證的 scope；**靜默失敗比報錯更危險**（handler 找不到元素跳過 → 旁白與畫面脫鉤），關鍵動作應在 handler 內驗證後果。
6. PII 檢查要覆蓋：畫面、log、git 歷史、原始素材四個面向。

## 13. Phase C 用戶 Panel 優化輪（完成）

依 /goal：3 輪 × 每輪 ≥3 位不同背景觀眾 agent × 各 5+ 建議 → 導演統整 → 優化工程。共 9 位 persona、逐幀觀影＋讀逐字稿、grounded 舉證。完整統整見 `video-assets/企劃/優化輪-導演統整.md`。

**Round 1（新手小資／當沖老手／存股波段，24 條）** → 導演分「立即可做／需重錄／需交易日／結構取捨」→ 執行 9 張新卡＋1 處旁白：成本損益兩平（3 族群共識）、跳動單位、零股規則、括號單 OCO、快捷鍵速查、殖利率填息、基差誤差、紅漲綠跌。

**Round 2（YouTube 創作者／CFP 金融講師／完全金融小白，看 R1 改進版）** → CFP 抓到**真正的正確性錯誤**（最高優先）：
- 選擇權角落 max/min 說成「最大賺賠」是錯的（買方獲利無上限）→ 改旁白＋補選擇權基礎卡＋買賣方風險卡
- T+2「後天」→「第 2 個交易日、遇假日順延」
- 委託狀態中英對照、期貨風險指標具體門檻
完全小白暴露基礎名詞缺口（K線紅綠棒、大盤、五檔掛單配對、漲跌停從沒解釋）→ 補 5 張基礎卡。B1：P8 釘台積電 2330、限價 2400/零股 80 股，讓零股範例「本金<一張」成立、且不再超漲停自相矛盾。共 12 張新卡＋7 處旁白＋P8 重錄。

**Round 3（R1 原三位回訪驗收，看最終版）** → 收斂：
- 小資新手 **SHIP**（5 項全修、"可上架"）｜當沖老手 **MINOR_FIXES**（3 項全補、1 個 LOW）｜存股波段 **SHIP**
- **三位零 regression**；驗收發現多處內部一致性（2400 是 5 元跳動整數、漲停2515=2290×1.1、零股 LMT+ROD 符合 P6 零股卡）
- 唯一 LOW（期貨當沖倉別/MKP 有畫面無解說）已補一句、並標「新手先跳過」以免與「小白被期貨搞混」衝突 → 三位等效 SHIP

**方法論收穫（Phase C）**：
1. 三輪換不同背景（先 3 種交易風格 → 再製作/教育/完全小白 → 再回訪驗收）比同一批看三次更能挖出不同層次的問題。
2. **CFP 抓到的正確性錯誤是自檢與一般 QC 都抓不到的**——需要領域專家 persona。教學片「講錯」比「不夠炫」嚴重得多。
3. 有些建議會**互相衝突**（當沖要更多期貨細節 vs 小白被期貨搞混）——導演要做取捨，用「標註可略過」化解，而非盲從單一 persona。
4. 每輪重組後**重產 transcripts**（否則下一輪讀到舊稿誤報）——已納入流程。
5. 回訪驗收（同一批看修正版）是確認「有沒有改壞」的關鍵一步，不能只單向收集建議。
