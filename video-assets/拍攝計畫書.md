## 一、核心架構（解決「講的跟畫面不一樣」）

**主題驅動、逐段成對**：每一段旁白綁定一個畫面動作，錄影器逐段執行並打時間戳，組裝器把每段畫面剪出來、精準對齊該段旁白。沒有循環湊時長，說什麼就演什麼。

```
topics_spec.py（旁白↔動作契約）
   → 錄影器 v3（逐段執行＋時間戳＋實測 bbox＋虛擬游標＋金框標註）
   → scenes.json（每段起迄＋zoom 座標）
   → 組裝器 v3（逐段剪輯對齊旁白＋zoom 裁切＋字卡插入＋字幕燒錄）
   → 七集成片 → QC agent 逐集驗片 → 交付
```

## 三、拍攝手法（參考影片格式）

1. **虛擬游標**＋永豐紅點擊漣漪、平滑滑行、點前停 0.3s（Playwright 原生不錄游標，必須注入）
2. **金框標註**：zoom 段在目標元素旁浮現 #D9A45B 金框＋標籤，座標用錄影時實測 bbox
3. **11 個 zoom 點位**已定座標策略（onboarding 三項固定座標、App 內段落錄時重抓）
4. **動作卡點**：點擊落在對應語句唸出的當下

## 四、視覺系統

- 字卡 18 張已生成核准樣式：片頭、7 章節卡、7 大綱卡、永豐申請步驟卡、本機監控警告卡、CTA

## 六、生產流程與品管

1. opus 實作 agent：錄影器 v3＋組裝器 v3（含游標/金框/zoom/分段對齊/字卡/雅黑正黑 Barlow 字幕系統）
2. **先拍 T1 試播集** → 你驗收（游標節奏/zoom 效果/同步感）→ 修正 → 批量拍 T2–T7
3. 每集 QC agent 逐格驗片（同步性/隱私/安全聲明/字幕），fail 就修根因重錄
4. 交付：七集 mp4 ＋ 可重跑管線（交易日重錄升級畫面生動度）
