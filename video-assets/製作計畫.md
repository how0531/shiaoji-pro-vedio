## 目標

做一支給交易者看的 Shioaji Pro 介紹影片，**要有實際操作的畫面**。主片 16:9 約 2.5 分鐘（YouTube/FB/官網），另出 9:16 短版約 40 秒（Threads/Reels/Shorts 導流）。

## 為什麼 Remotion 是「做影片」的答案

Remotion 用 React 程式碼產出真正的 .mp4，這正是「能做影片的 skill」。它能：
- 用 `<OffthreadVideo>` **嵌入你的 Playwright 操作錄影**當底層畫面
- 疊上片頭字卡、功能字幕條（lower-third）、轉場、進度動畫
- 用 `<Audio>` 放旁白與配樂
- `remotion-captions` skill 可用 Whisper **自動把旁白轉成字幕**燒進畫面
- `npx remotion render` 一鍵輸出 mp4

官方 skill 包 `remotion-dev/skills` 含：remotion-create（建專案）、remotion-captions（字幕/轉錄）、remotion-render（渲染）、remotion-markup / remotion-best-practices（排版規則）、mediabunny（讀取媒體資訊）等。

> ⚠️ 這些 skill **目前沒裝在協作環境裡**，我無法用 `/skill` 直接叫。兩條路：
> **(A) 你安裝** remotion-dev/skills 到你的 Claude Code（plugin/skills 目錄），我就能照它的官方排版與字幕最佳實踐幫你把整支影片建起來、渲染。
> **(B) 不裝也行**：Remotion 我熟，可直接把 composition 程式碼寫進 `video-assets/remotion/src/`，你本機 `npm i && npx remotion studio` 預覽、`npx remotion render` 輸出。

## 建議的完整流程（本機執行）

```sh
# 1) 錄實際操作畫面（模擬環境，不動真錢）
shioaji server start                 # 模擬環境 :8080
pnpm dev                             # App :5173
uv run --with playwright python3 scripts/capture-demo-video.py
ffmpeg -i docs/demo-raw/*.webm -c:v libx264 -crf 18 -pix_fmt yuv420p \
  video-assets/remotion/public/demo.mp4

# 2) 配音（擇一）
#   - AI TTS：把 影片腳本與分鏡.md 的「配音稿」貼到 剪映/CapCut 朗讀、ElevenLabs 或 Azure TTS
#   - 自己錄：對著配音稿念
#   產出 voiceover.mp3，放到 video-assets/remotion/public/

# 3) 合成（Remotion）
cd video-assets/remotion
npx create-video@latest --yes --blank --no-tailwind .   # 首次：建立專案骨架
# 放入我寫的 src/DemoVideo.tsx 等 composition 檔
npm i
npx remotion studio                  # 預覽、微調
npx remotion render DemoVideo out/shioaji-pro.mp4   # 輸出成片
```

## 配音／剪輯的替代工具（若不走 Remotion）

- **剪映 / CapCut（推薦給中文）**：內建 AI 螢幕錄影＋中文語音辨識（96–98% 準確）＋自動字幕＋朗讀配音，一站到底，最省事。
- **Arcade / Guidde**：錄一次操作，自動生成 demo 影片＋AI 旁白＋字幕，適合快速產出。
- **HeyGen / Higgsfield MCP**：可從 Claude 直接生成 AI 影片／虛擬主播，但比較適合「解說型」而非「真實產品操作」畫面。

## 誠實紅線（影片與貼文都要守）

- 說「介面 100% 開源」，不要說「全部開源」（桌面殼層／AI／回測是閉源）
- 停損停利標「客戶端觸價，App 關了不觸發」
- 需永豐帳戶；預設模擬環境，切正式是真錢
- AI Agent／策略回測是桌面版限定，別暗示 Web 版可用
