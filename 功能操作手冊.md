## 共通操作（先讀這段，後面每則就不再重複）

終端的畫面是一面可自由拼貼的「面板牆」——每個功能都是一塊浮在 React Grid Layout 格線上的面板（Block）。以下操作在幾乎每個面板都通用：

**如何新增面板：** 在頂部導覽列（HudHeader）右側點「**＋ 新增面板**」選單，會展開標題為「新增面板 Add Panel」的下拉，裡面列出全部 24 種面板，點任一項即在工作區最底部新增。這是唯一入口——沒有拖曳新增、也沒有右鍵選單。若某面板是「單例」（如自選清單、排行榜、通知中心），且工作區已有一個，選單中該項會變灰並標「（已存在）」。

**標題列（PanelChrome）上的控制項：**
- **拖曳把手：** 整條標題列就是把手，按住拖曳即可在格線上搬移；標題列上的按鈕、輸入框、下拉不會觸發拖曳。
- **連動 ↔ 鎖定（僅「可鎖定」面板有）：** 預設是「連動」（鏈結圖示 Link2），面板跟隨您在自選清單／⌘K／排行／T字／熱力圖選定的全域商品；點一下切成「鎖定」（圖釘 Pin），面板固定顯示某一檔，此時會出現代碼輸入框可直接改代碼（Enter 套用）。再點一下回到連動。
- **彈出獨立視窗（僅 11 種面板有）：** 外連圖示（ExternalLink），把面板轉成獨立 OS 視窗丟到第二螢幕。支援彈出的是：K 線圖、五檔、下單面板、成交明細、閃電下單、籌碼資訊、分價量表、行情回放、委託簿熱圖、選擇權 T 字、損益分析。
- **關閉：** 最右的 ✕，移除面板並即時存檔。

**頂部導覽列（HudHeader）上的常駐功能：** 環境徽章（模擬環境／正式環境）、大盤狀態列、串流狀態燈（LIVE／SYNC／LOST）、帳號選單（證券 S／期貨 F 切換）、**風控**選單（Kill Switch）、**＋ 新增面板**、**⚡ 全開**（自選前 N 檔一次開多張閃電下單）、**版面**選單（預設版面／命名儲存／載入）、**主題**選單（配色／字級／隱私遮蔽／音效）、以及伺服器管理（桌面版）。

**全域快捷鍵：**
- **⌘K／Ctrl+K** — 商品搜尋跳轉（支援中文股名，即使正在輸入框也有效）
- **Esc Esc**（0.6 秒內連按兩次）— 撤銷所有未成交委託（第一次只提示）
- **B／S** — 下單面板切買進／賣出（非輸入狀態下）

**連動 vs 鎖定：** 每個面板有 pin：連動（pin=null）＝跟隨全域選取商品；鎖定（pin=代碼）＝固定某一檔。只有會吃商品的面板（K 線圖、五檔、下單、成交明細、閃電、籌碼、分價量表、行情回放、委託簿熱圖、鋪單）才有這個切換；自選清單／排行／T字／熱力圖本身是「選商品的來源」，損益／帳務面板則顯示全帳戶資料，都不需要這個切換。

**版面：** 內建多組預設版型（標準看盤、當沖交易、雙圖對照、選擇權、鋪單交易、閃電矩陣、熱力選股、AI 副駕、分析研究…），可一鍵套用；也能把自己擺好的版面命名儲存／載入。所有設定都存在本機瀏覽器，隨開隨用、自動還原。

## 行情看盤

### 自選清單 Watchlist

在 Shioaji Pro 交易終端中，「自選清單（Watchlist）」是您整個看盤動線的起點。它不只是一份代碼列表——在這個終端裡，自選清單同時扮演「全域選取商品的來源」：您在清單裡點哪一檔，K 線圖、五檔、成交明細、下單面板等所有處於「連動」狀態的面板就會一起跳到那一檔。清單本身存放在後端（server-backed），支援多份清單切換，任何新增、刪除、拖曳排序都會即時寫回伺服器；每一列的報價來自即時報價串流（useQuote hook），漲跌幅則是以「最新價與參考價（昨收）」重新換算，而非直接採用串流欄位，以避開股票與期權串流單位不一致的問題。若您開啟迷你走勢圖，每列右側會用一張 Canvas 小線圖（Sparkline 元件，資料取自當日 1 分 K）畫出當日走勢，並以昨收畫一條虛線當基準。

以下為您說明如何開啟與操作這個功能：

1. 如何開啟「自選清單」面板
- **預設已在版面上：** 自選清單屬於預設工作區（DEFAULT_WORKSPACE）的左欄，通常一開啟終端就在那裡，多數情況您不需要另外新增。
- **重新新增：** 若您把它關掉了，點頂部列（HudHeader）右側的「＋ 新增面板」選單，在「新增面板 Add Panel」清單中選「自選清單」。請注意它是單例（singleton）面板，全終端只能有一個；若已存在，選單裡的「自選清單」會呈灰色並標「（已存在）」，點不動。
- **無法彈出：** 自選清單不在可彈出（POPOUT_TYPES）名單內，標題列沒有彈出鈕，無法轉為獨立視窗。

2. 點選切換全域商品（連動整個終端）
- **它是選取來源，不是跟隨者：** 與 K 線圖、五檔那類面板不同，自選清單本身「不可鎖定」，標題列沒有連動↔鎖定切換鈕——因為它就是全域選取的源頭。
- **點一列＝設定全域商品：** 點任一列即把該商品設為全域選取（selected），所有處於「連動」狀態的面板會一起切換過去；目前選取的那一列會反白標示。

3. 新增商品到清單
- **底部輸入框：** 面板最下方的輸入框（提示文字為「股票、期貨或指數（如 台積電期）」），輸入代碼或名稱即可，系統會自動判斷商品類型。
- **即時建議：** 輸入後約 150ms 會跳出建議清單，逐筆顯示代碼／名稱／類別，點任一筆直接加入。
- **直接送出：** 也可按 Enter 或右側「＋」鈕加入（會自動去空白、轉大寫）。清單為空時，面板會提示「清單是空的 — 在下方輸入代碼加入」。

4. 移除商品
- **滑到該列點 ✕：** 每列右側有一個小 ✕ 鈕（提示「從清單移除」），點它即把該商品從目前清單移除，並同步寫回伺服器。

5. 拖曳排序（自訂順序）
- **直接拖曳整列：** 在「自訂順序」模式下，用滑鼠拖曳整列上下調整位置，放開即完成，新順序會即時存回伺服器。
- **排序模式下暫停：** 一旦切到漲跌幅排序，拖曳排序會停用（只有自訂順序才能手動排列）。

6. 依漲跌幅排序
- **排序鈕循環三態：** 標題列的排序鈕（雙箭頭圖示）點第一下切「漲幅在前」（向下箭頭），第二下切「跌幅在前」（向上箭頭），第三下回到「自訂順序」。
- **每 10 秒重排：** 排序開啟時，系統會每 10 秒依最新漲跌幅重新排一次，盤中強弱一目了然。

7. 迷你走勢圖開關
- **趨勢圖示鈕：** 點標題列的趨勢線圖示鈕即可開／關每列的迷你走勢圖，這個偏好會記在本機（localStorage，鍵 `sj-pro-watchlist-spark`），下次開啟仍記得。
- **當日 1 分 K ＋ 虛線基準：** 小線圖以當日 1 分 K 收盤點連成，昨收（參考價）畫成虛線基準線並決定線色（在基準之上為漲色、之下為跌色）；同一檔的資料以模組級快取共用，不會因為多列而重複打 API。

8. 成交閃動提示
- **真成交才閃：** 當某檔有真實撮合成交時，該列會依方向（紅漲綠跌）閃動一下。這個閃動只在「真實成交」時觸發，盤前／盤中的試撮不會閃，方便您用餘光察覺哪幾檔剛剛有量進出。

9. 管理多份清單
- **切換清單：** 標題列左側的下拉選單列出所有清單，後綴會顯示各清單檔數（如「主力股（12）」）。
- **重新命名：** 點鉛筆鈕，該列會就地變成輸入框，Enter 或點到別處即確認、Esc 取消；若改成重複的名稱會被拒絕並停在編輯狀態讓您修正。
- **新增清單：** 點「＋」鈕，輸入新清單名稱後按「建立」。
- **刪除清單（兩段式防呆）：** 點垃圾桶鈕會先變成「確認?」，需在約 2.5 秒內再點一次才會真的刪除整份清單，逾時自動取消，避免誤觸。

### K 線圖與圖表交易

在 Shioaji Pro 交易終端中，「K 線圖」面板不只是看盤工具，它把「畫圖」與「下單」合而為一——您在圖上看到的每一個價位，都可以直接點成一筆委託、一條停損停利，或一個到價提醒。它的 K 棒資料來自 Shioaji 的 kbars 歷史 API（原生只提供 1 分鐘棒，較大的週期由前端即時聚合而成），繪圖引擎採用 lightweight-charts v5，盤中則靠 SSE 行情串流（useQuote）逐筆更新最後一根 K 棒。整個面板由上方的 QuoteBoard 報價板與下方的 CandleChart 組成，是可以同時開多張的核心看盤面板。

以下為您說明如何開啟與操作這個功能：

**1. 如何開啟「K 線圖」面板**

**新增面板：** 在交易終端頂部列（HudHeader）右側，點擊「＋ 新增面板」選單，於「新增面板 Add Panel」清單中選擇「K 線圖」，系統就會在您的工作區（React Grid Layout）底部新增一張圖表面板。

**可開多張：** K 線圖不是單例面板，您可以新增多張，例如一張看台指期、一張看台積電，或搭配「雙圖對照」版面預設（一張連動、一張鎖定 TXFR1）同時觀察。

**彈出獨立視窗（可選）：** 點面板標題列右上角的彈出鈕（ExternalLink 圖示），可把這張圖轉成獨立視窗放到第二螢幕，專心盯盤。

**連動或鎖定商品：** 這張圖預設「連動」（標題列的 Link2 連動鈕），會跟隨您在自選清單／⌘K 搜尋／排行榜點選的全域商品；若您想讓它固定盯某一檔，點一下把它切成「鎖定」（Pin 圖示），或在標題列的代碼框輸入代碼後按 Enter，之後這張圖就只顯示該商品。

**2. 切換週期與回溯歷史**

**五種週期：** 圖表左上角工具列有 1m、5m、15m、60m、1D 五個週期鈕，預設載入 5m。點一下即切換，切換後會自動捲到最新一根並重新自動縮放。

**歷史回溯：** 想看更早的走勢，直接把圖往左拖，接近左緣時系統會自動再向上游抓一頁更舊的 kbars 接上去（TradingView 式無限捲動），可一路回溯到約三年前的資料為止；連續抓不到資料時就會停止。

**重設視圖：** 若您手動拖過價格軸或平移到角落，導致畫面跑掉，點週期鈕右邊的「重設視圖」鈕（展開／全螢幕式圖示（四角外擴箭頭），提示為「重設視圖（自動縮放）」），即可重新自動縮放、把 K 棒完整帶回畫面。

**3. crosshair 十字游標與圖例（legend）**

**滑鼠讀值：** 把游標移到圖上，十字游標會標出該時間點的價位；若您掛了技術指標，左上角圖例（legend）會即時顯示游標所在那一根 K 棒的指標數值，游標移開時則回到顯示最新一根的數值。

**同步下單價：** 移動游標的同時，圖表會把游標對應的價位（自動吸附到該商品的最小跳動單位）同步給「下單面板」，方便您以圖上價位帶入委託價。

**技術指標：** 工具列右側的「指標」鈕會開啟指標選單，加入的疊加型指標畫在主圖、震盪型指標各自獨立成一個副圖 pane；每條指標在圖例上都有隱藏（眼睛）、設定、移除與「更多」（複製、上下移調整疊圖順序、加入我的最愛）等控制。

**4. 圖上點價交易（六種交易模式）**

工具列中段有一整排交易模式鈕，切換後圖面上方會出現對應的提示文字。除「游標」外，每種模式都是「一次性」的——每點一次圖、送出一筆後就自動跳回「游標」，避免誤點連續下單。旁邊的「量」欄位設定這些動作要用的口數／張數（預設 1）。

**游標（observe）：** 預設模式。點圖只會把該價位挑選並同步給下單面板，不會送出任何委託，適合純看盤或替下單面板選價。

**點價買 / 點價賣：** 切成此模式後，在圖上點一個價位，就會以該價位掛出一筆限價買進／賣出委託並立即送出。請注意這是「真實委託、點一下即送出、沒有二次確認」，下單前務必先確認模式與數量；一般點價買賣仍受頂部「風控」選單的 Kill Switch（鎖定下單）約束。

**停損 / 停利：** 切成此模式後在圖上點一個價位，會在該價位掛一條觸價單，方向由「點擊價位相對目前現價的上下」自動判斷（點在現價下方視為向下觸發、上方為向上觸發）。觸價後會以市價送出對應的反向平倉單。停損／停利觸價單屬於保護性出場，特別設計為「不受風控鎖定封鎖」，即使 Kill Switch 已鎖定仍能觸發送出。

**警示（到價警示）：** 在圖上點一個價位設定到價提醒，觸價時只跳通知、不下任何單。

**5. 委託線與觸價線（圖上管理你的單）**

**委託線 overlay：** 您在這檔商品上所有未成交的委託，都會以實線畫在圖上（買單用漲色、賣單用跌色，線上標示「買／賣＋剩餘量」）。

**拖曳改價：** 把滑鼠移到委託線附近，游標會變成上下箭頭（ns-resize），按住往上下拖到新價位放開，系統就會送出改價（自動吸附最小跳動單位），成功後跳出「改價已送出」通知。

**CANCEL 刪單：** 圖左上角的清單會列出每一筆委託（顯示「委買／委賣＋量＠價」），點該列的「CANCEL」即可撤銷該筆委託。

**觸價線（虛線）：** 您掛的停損、停利、到價警示會以虛線畫在圖上——停損為琥珀色、停利為十字游標色、警示為灰色，並在軸上標出價位與「停損買/賣、停利買/賣、警示」等字樣。同一份清單裡每個觸價／警示項目後面都有一個小叉，點它即可移除。

**6. 重要提醒：觸價單與警示是「客戶端」監控**

圖上掛的停損、停利、到價警示是由前端的觸價引擎（trigger-engine）在本機監看 SSE 行情串流達成的。它們雖然會存進瀏覽器（localStorage）而不會因重新整理就消失，但**只在本 App 開啟、且行情連線正常時才會運作；一旦您關閉程式或斷線，這些觸價單與警示就不會被監控、也不會觸發**。若需要盤中長時間離開，請改用券商伺服器端的委託方式，切勿把重要的停損完全託付給圖上的觸價單。

### 技術指標與自訂指標

在 Shioaji Pro 交易終端裡，「技術指標」不是一個獨立面板，而是內建在 **K 線圖（CandleChart）** 上的一套疊圖引擎。它把即時／歷史 K 棒資料交給 21 種內建指標公式運算後，直接畫在主圖上（如均線、布林通道），或另開一格獨立副圖窗格（如 MACD、RSI、KD），全部在您本機端計算、隨 K 棒逐根對齊。選擇器本身刻意做成 TradingView 式的介面（`IndicatorDialog`）：左側分類、上方搜尋、我的最愛，點一下指標會先跳出設定視窗（`IndicatorSettingsModal`）在圖上即時預覽，按「確定」才真正加入。若內建指標不夠用，還有一個 in-app 的自訂指標編輯器（`CustomIndicatorEditor`），讓您用 JavaScript 搭配 `ta` 函式庫寫自己的公式，存檔前會先送進 Web Worker 沙箱驗證（2 秒逾時，擋掉無窮迴圈），並自動偵測您 `plot()`／`hline()` 宣告的輸出。

以下為您說明如何開啟與操作這個功能：

1. **如何開啟指標選擇器：** 技術指標掛在 K 線圖上，所以請先有一張 K 線圖面板——在頂部列點「＋ 新增面板」選單，選「K 線圖」加入一張圖。接著在該圖表工具列上點「**指標**」按鈕，就會彈出 TradingView 式的「技術指標」選擇器。（當這張圖已經掛了指標時，這顆按鈕會呈現高亮狀態。）

2. **搜尋與分類瀏覽：** 選擇器上方是搜尋框，輸入名稱、縮寫或中英文關鍵字都能過濾（例如打「rsi」「均線」「布林」）。左側分類側欄可切換「全部指標／我的最愛／主圖疊加／副圖指標／自訂指標」五類。每一列指標右側都有一顆星，點下去即加入「我的最愛」（會記住，下次直接在最愛分類找得到）；已經加到圖上的指標，列上會標「已加入 N」。

3. **21 種內建指標，加入前先預覽：** 清單分成「主圖疊加」（SMA、EMA、WMA、布林通道、VWAP、SAR、Supertrend、Donchian、Keltner）與「副圖指標」（MACD、RSI、KD、Stoch RSI、CCI、ATR、OBV、MFI、Williams %R、DMI、ROC、乖離率）兩段，共 21 種。**點任一列指標**，不會馬上加進去，而是先開啟該指標的設定視窗、同時在圖上即時預覽；您調到滿意按「確定」才算加入，按「取消」則整個撤掉。同一種指標可以重複加多個（例如兩條不同週期的 EMA），彼此參數各自獨立。

4. **指標設定視窗（輸入／樣式／時框顯示）：** 設定視窗最多分三個分頁。**輸入：** 調整週期等參數（有最小／最大範圍限制）。**樣式：** 每一條輸出線都能各自開關顯示、換顏色與線型——點顏色鈕會就地展開色盤（灰階列＋十色相多明度，另可直接填 `#rrggbb`、拉透明度 10～100%、選 1～4px 粗細）；點線型鈕可切「線／階梯線／面積／柱狀／圓點」。下方「輸出數值」還能設定小數位數（自動或 0～4 位）、是否在價格軸顯示最新值標籤、是否在圖上顯示數值。**時框顯示：** 勾選這個指標要在哪些時間週期出現（例如只在日線顯示、分線不顯示）。

5. **存成我的預設：** 設定視窗底部有「預設值」下拉，可「重設為內建預設」把這個實例還原；也可以「存為我的預設」——之後每次新增同一種指標，就會直接套用您這組色彩與參數（會跳「已存」提示）。旁邊的「移除」則把這個指標從圖上刪掉。

6. **副圖獨立窗格：** 屬於「副圖指標」類的（MACD、RSI…）不會擠在主圖上，而是各自佔一格獨立的下方窗格，legend（圖例）疊在自己那格的左上角；點該 legend 名稱即可再次打開它的設定。主圖疊加類（均線、布林…）則直接畫在 K 棒上。

7. **寫自己的自訂指標：** 在選擇器左下點「建立自訂指標」開啟編輯器。先填名稱、legend 縮寫、選「主圖疊加或副圖」、（選填）一句說明。**參數：** 用「加參數」建立可調參數（代號、顯示名稱、預設、最小、最大、間距），程式碼裡用 `p.代號` 取值。**程式碼：** 在下方文字框用 JavaScript 搭配 `ta` 函式庫撰寫，內建 `open/high/low/close/volume/time/hl2/hlc3/ohlc4` 等與 K 棒對齊的序列，以及 `ta.sma/ema/rsi/atr/crossover…` 等函式；用 `plot('名稱', 序列, { kind, color })` 宣告輸出、用 `hline(數值)` 畫參考水平線（旁邊「語法說明」可隨時展開速查表）。

8. **驗證、微調輸出、儲存：** 寫完點「驗證」——程式會送進 Web Worker 沙箱試跑（2 秒逾時保護，避免寫錯迴圈卡住），通過後會顯示「偵測到 N 條輸出、水平線 …」，並列出每一條 `plot()` 讓您微調顯示名稱、線型（線／虛線／柱狀／圓點）與顏色。改過程式碼要重新驗證才能存。按「儲存」後，這個自訂指標就會出現在選擇器的「自訂指標」分類，用法與內建指標完全一樣（列上會標「自訂」，可用鉛筆再編輯、或在編輯器內「刪除指標」）。

9. **跟隨商品（連動）或鎖定，以及彈出第二螢幕：** K 線圖是可連動、可鎖定的面板：連動時整張圖（含所有指標）跟著您在自選清單／⌘K／排行榜等處選的商品走；點標題列的連動鈕改成鎖定，就把這張圖與其上的指標固定在某一檔。K 線圖也支援彈出——點面板右上的彈出鈕可把整張含指標的圖轉成獨立視窗，丟到第二螢幕專心盯盤。

（提醒：所有指標運算都在您本機端即時計算，屬看盤輔助資訊，不會下單，也不會在關掉程式後於雲端持續運作。）

### 五檔報價與委託簿熱圖

在 Shioaji Pro 交易終端中，「五檔（Depth）」與「委託簿熱圖（Depth Map）」是一組互補的盤口工具，專為判讀當下買賣掛單力道、抓進出場價位而設計。兩者都吃 Shioaji API 推送的五檔委託（bidask）資料，但呈現方式不同：**五檔面板（DepthLadder）**用 `useQuote` 訂閱即時報價，把買一到買五、賣一到賣五的委託量畫成左右對照的量能條，讓您一眼看出哪一檔掛得厚、目前買賣力道偏哪邊；**委託簿熱圖（DepthMap）**則用 `subscribeQuoteStore` 持續累積每一次五檔更新，在一張 HTML canvas 上以顏色深淺重建掛單牆的時間序列——橫軸是時間、縱軸是價格、越亮代表那個價位的掛單越厚——用來觀察掛單牆的堆疊與撤單。

以下為您說明如何開啟與操作這個功能：

**1. 新增「五檔」面板：** 在交易終端頂部導覽列（HudHeader）右側，點擊「＋ 新增面板」選單，於彈出的「新增面板 Add Panel」清單中選擇 **「五檔」**。系統會在您目前的工作區（React Grid Layout）最底部新增一個五檔面板。此面板非單例，需要對照多檔商品時可以開多個。

**2. 讀懂量能條：** 面板分成「買量 / BID / ASK / 賣量」四欄，共五列，對應買一～買五與賣一～賣五。每一檔的量能條長度會依「這五檔裡的最大單量」等比例縮放，所以條子越長代表該價位掛單相對越多。面板底部一列顯示「Σ買（五檔買量總和）」、「價差（買一賣一之差）」與「Σ賣（五檔賣量總和）」；最下方還有一條**買賣力道橫條**，用買方量佔總量的百分比（買 X%）顯示當前五檔的多空傾向。

**3. 點價帶入下單：** 這是五檔面板最實用的操作——直接用滑鼠**點任一檔的買價或賣價色條**，該價格會透過內部 price-sync 機制帶入同一商品的下單面板（OrderTicket），省去手動輸入。請注意它**只是把價格填進下單面板、不會直接送出委託**，真正下單仍要在下單面板確認，因此建議把「五檔」和「下單面板」放在一起使用。

**4. 連動或鎖定商品：** 五檔面板預設為**連動**狀態，會跟隨您在自選清單、⌘K 搜尋或排行榜選取的全域商品。若想讓它固定盯著某一檔，點面板標題列（PanelChrome）上的連動鈕即可**鎖定**目前商品；鎖定後標題列會出現代碼輸入框，可直接改代碼、按 Enter 套用，再點鎖定鈕即恢復連動。

**5. 新增「委託簿熱圖」面板：** 同樣從頂部「＋ 新增面板」選單中選擇 **「委託簿熱圖」**。它一樣可開多個、預設連動當前選取商品，也可用標題列連動／鎖定鈕切換到固定商品。

**6. 解讀熱圖與注意其記錄方式：** 熱圖以橫軸為時間、縱軸為價格作圖，顏色亮度代表該價位的掛單厚度——買方掛單用漲色、賣方掛單用跌色，中間那條細線是最新成交價的走勢，方便您觀察掛單牆在哪些價位堆積、又在何時被撤走。**重要提醒：此面板是「開啟之後才開始即時累積記錄」**（面板下方也標註「開啟後開始記錄」），它繪製的是程式執行期間逐步收集的資料，並非回補的歷史；因此**一旦關閉面板或切換商品，先前累積的熱圖會清空並從頭記錄**，想長時間觀察某檔請保持面板開著、不要中途切換。

**7. 彈出為獨立視窗：** 「五檔」與「委託簿熱圖」都支援彈出（屬於 POPOUT_TYPES）。若您使用多螢幕，可點面板右上角的彈出鈕，把面板轉成獨立視窗放到第二螢幕上專心盯盤。

**8. 指數不支援：** 大盤指數（IND）沒有五檔委託資料，因此在五檔與委託簿熱圖面板選到指數時，面板不會顯示盤口，而是改以不支援訊息（IndexBlockUnavailable）提示您改看個股、期貨或選擇權等有五檔的商品。

### 成交明細與分價量表

在 Shioaji Pro 交易終端中，「成交明細（Tape）」與「分價量表（Volume Profile）」是兩張搭配使用的盤中微結構面板，幫您把「每一筆成交」與「每個價位的買賣力道」看清楚。它們都以 Shioaji API 取得的當日歷史成交 Tick 作為起點：面板一開就先把今日已成交的資料補齊，接著透過即時 Tick 串流（onAnyTick）把新成交疊上來，不需要您手動整理資料。兩張面板都是可鎖定的看盤面板（PanelChrome 標題列提供連動／鎖定切換），並放在您工作區的 React Grid Layout 版面裡，可自由搬移、縮放，也可彈出成獨立視窗。

成交明細（TickTape）重建的是「時間軸」：由上而下、最新在最上，逐筆列出成交時間、成交價與成交量，時間精確到微秒（HH:MM:SS.ffffff）。分價量表（VolProfile）重建的是「價格軸」：把當日成交量依價位彙總，畫成一根根橫向長條，並在頂部給您這檔的內外盤比。

以下為您說明如何開啟與操作這個功能：

**新增面板：** 在交易終端頂部列（HudHeader）右側點「＋ 新增面板」選單，於「新增面板 Add Panel」清單中選擇「成交明細」或「分價量表」，系統就會在您目前的工作區底部新增該面板。這兩種都不是單例面板，您可以同時開好幾個（例如一張鎖台積電、一張鎖台指期）對照著看。

**它跟著哪一檔（連動）或鎖定某檔：** 兩張面板預設都是「連動」狀態，會跟隨您在終端裡目前選取的商品（由自選清單、⌘K 商品搜尋、排行榜或熱力圖等任一路徑選定）。若您想把某張面板固定在特定商品上，點面板標題列的連動鈕（Link2 圖示）即可鎖定目前商品；鎖定後標題列會出現代碼輸入框，可直接輸入代碼（如 2330、TXFR1）並按 Enter 套用，再點鎖定鈕（Pin 圖示）即可恢復連動。

**成交明細怎麼看：** 面板一開啟就會載入今日最近的歷史成交（約最新 120 筆），面板中央會短暫顯示「載入歷史成交…」占位提示；補齊後即時成交會從最上方持續插入。每一列由左至右是「時間、成交價、成交量」三欄：時間顯示到微秒；成交價會依買賣方向上色（成交在賣方掛單、屬外盤買進的走漲色，成交在買方掛單、屬內盤賣出的走跌色）。當累積筆數足夠後，系統會以近期成交量的滾動平均為基準，把明顯偏大的單筆成交（約達平均量三倍以上）以較醒目的樣式標示出來，方便您一眼抓到大單進出。

**分價量表怎麼看：** 面板頂部是「內外盤比」——把當日累積的外盤（成交在賣方價、tick_type 買方）與內盤（成交在買方價、tick_type 賣方）換算成百分比，並用一條左右比例條呈現，方向不明的成交會各計一半。下方主體是「分價量」：每個價位一列，由高價到低價排序，橫條同時畫出該價位的外盤（買）與內盤（賣）量，右側標示該價位總量；為了可讀性，價位很多時會只保留成交量最大的前四十個價位。它同樣是開啟即先統計今日歷史 Tick（統計中顯示「統計分價量中…」），之後再把即時成交累加進去。

**指數（IND）的限制：** 大盤指數沒有逐筆成交 Tick，因此這兩張面板對指數不支援盤中更新——選到指數時，面板會顯示對應的不支援說明，而非成交列表或分價長條。若要看逐筆與分價，請鎖定或連動到個股、期貨或選擇權等有 Tick 的商品。

**彈出為獨立視窗（多螢幕）：** 成交明細與分價量表都支援彈出。點面板標題列右上的彈出鈕（ExternalLink 圖示），即可把該面板轉成獨立視窗放到第二螢幕，專心盯逐筆與分價（彈出視窗會採用較長的輪詢間隔以避免多視窗同時打爆上游限流）。這兩張都是純看盤的資料面板，不含下單或設定按鈕，所有操作集中在標題列的連動／鎖定、彈出與關閉。

### 報價看板與權證

在「Shioaji Pro」交易終端中，「報價看板」與「權證篩選器」是兩個互補的行情看盤工具。「報價看板」（QuoteBoard）本身並不是一個可獨立新增的面板，而是內建在「K 線圖」面板頂端、K 線之上的一整排大字報價板；它透過 useQuote 串流訂閱您當前商品的即時 Tick／指數／五檔資料，把成交價、漲跌、開高低量與最佳買賣一檔攤在圖表上方，讓您在看圖的同時一眼掌握盤面。「權證篩選器」（WarrantPanel）則是一個獨立的單例面板，它以一檔標的股為核心，向後端撈出該標的所有發行中的權證，依價內外或到期日排序，方便您挑到合適的權證後連動整個終端或加入自選。

以下為您說明如何開啟與操作這個功能：

1. **報價看板跟著「K 線圖」一起出現：** 報價看板不在「＋ 新增面板」清單裡，無法單獨新增。您只要在頂部導覽列（HudHeader）的「＋ 新增面板」選單開一個「K 線圖」面板，看板就會固定顯示在該面板頂端。

2. **報價看板顯示的內容：** 左側是商品代碼與名稱，中央是依漲跌紅綠上色的大字成交價，觸及漲跌停時會加註「漲停／跌停」標記，其下並列漲跌點與漲跌幅。（漲跌幅是以漲跌點除以參考價換算，刻意不直接採用串流的 pct 欄位，以避免股票與期貨單位不一致。）再下方的資訊格，一般股票／期貨會列出開、高、低、量，以及參考價、漲停價、跌停價、時間與委買／買量／委賣／賣量的最佳一檔；若當前商品是大盤指數（IND），則改列上漲／平盤／下跌家數與漲停／未成交／跌停家數。

3. **報價看板的連動與鎖定：** 因為它是「K 線圖」面板的一部分，跟隨的商品完全由該 K 線圖面板決定——在 K 線圖標題列（PanelChrome）點「連動」鈕即跟隨全域選取商品，或點一下鎖定成固定代碼；把 K 線圖彈出為獨立視窗時，報價看板也會一起出現在彈出視窗中。看板自己沒有獨立的開關或彈出鈕，另外整條報價看板同時就是該面板的拖曳把手（drag-handle）。

4. **如何開啟「權證篩選器」：** 在頂部導覽列（HudHeader）點「＋ 新增面板」，於「新增面板 Add Panel」清單中選「權證篩選器」，系統即在您的工作區（React Grid Layout）底部新增此面板。它是單例面板，若工作區已有一個，該選項會標示「（已存在）」而點不動。此面板本身是「選標的的來源」，因此標題列沒有連動／鎖定切換，也沒有彈出獨立視窗的按鈕。

5. **挑選標的股：** 面板左上角的標的選擇器（UnderlyingPicker）用來指定要看哪一檔股票的權證，預設帶入台積電 2330（或您上次挑選並被記住的標的）。換標的後，系統會重新撈出該標的所有發行中的權證，並抓取一次現貨報價。

6. **篩選與排序：** 工具列提供三組控制——「全部／認購／認售」切換要看的權證類型；「到期區間」下拉可選 90 天內、180 天內（預設）、一年內或全部到期日；「排序」下拉可在「價內外排序」（最接近價平者排前）與「到期日排序」之間切換。上方摘要列會顯示標的代碼與名稱、現貨價、發行中權證檔數，以及目前符合條件顯示的檔數（最多列出最接近條件的 60 檔）。

7. **看懂列表欄位：** 每一列由左到右是權證代碼與名稱、類型（認購／認售）、成交價（依漲跌上色）、履約價、價內外（以現貨與履約價估算的百分比，標示「價內」或「價外」）與到期日（含剩餘天數）。

8. **點列連動、星號加入自選：** 點任一列即把整個終端連動到該權證，K 線圖、五檔、下單面板等連動面板都會一起切到這檔權證；每列最右的星號（Star）按鈕則是把該權證加入您目前的自選清單，成功會跳出「已加入自選」通知。

9. **資料更新方式：** 權證的成交報價是以每 8 秒輪詢一次快照（snapshot）的方式更新，並非逐筆即時 Tick，因此數字會以數秒為間隔跳動。載入中、載入失敗與查無符合條件時，面板會分別顯示「載入權證市場…」「權證資料載入失敗」與「目前篩選條件沒有權證」。

### 大盤狀態列

在 Shioaji Pro 交易終端中，「大盤狀態列（MarketBar）」是為了讓您在盯盤的任何時刻，都能一眼掌握大盤方向與台指期基差而設計的。它把「加權指數」的即時價與漲跌幅、以及「台指期近月 − 加權指數」的基差（正逆價差），濃縮成兩枚小徽章（chip），常駐在畫面最上方，讓您不必額外開面板就能隨時判斷多空氣氛與期現貨的價差結構。技術上，它是一個獨立的 `<MarketBar />` 元件，直接掛在頂部導覽列（HudHeader）裡；資料來源同時走兩條線：每 10 秒向 Shioaji 輪詢一次 `IX0001`（加權指數）與 `TXFR1`（台指期近月）的快照（fetchSnapshots），再疊上即時報價串流（useQuote）的最新成交，讓數字在盤中持續跳動；顏色則套用 `panel.dirText` 的漲/跌/平三態，會跟隨您在「主題」選單設定的漲跌配色（紅漲綠跌或綠漲紅跌）。

以下為您說明如何開啟與操作這個功能：

**免開啟、頂部列常駐：** 大盤狀態列不是可新增的面板，您不需要透過「＋ 新增面板」去把它叫出來。它固定掛在頂部列（HudHeader）上，只要終端啟動、後端有回傳大盤資料，它就會自動顯示，也不會被關閉或移除。

**加權指數（加權 chip）：** 左邊那枚標著「加權」的徽章，顯示加權指數（IX0001）的最新點數與漲跌幅。點數優先取自即時串流的收盤，沒有串流資料時退回 10 秒輪詢的快照；漲跌幅以基準價（前一參考價）換算，並依方向上漲、下跌或持平呈現對應顏色。

**台指期基差（基差 chip）：** 右邊那枚標著「基差」的徽章，計算的是「台指期近月（TXFR1）成交價 − 加權指數」，也就是期現貨的價差。正值代表期貨在指數之上（正價差）、負值代表逆價差，數字同樣依方向著色。把滑鼠移到這枚徽章上，會浮出提示文字「台指期 − 加權指數（價差）」，提醒您這個數字的定義。

**固定追蹤，不跟隨選取商品：** 這條狀態列永遠盯著「加權指數」與「台指期近月」兩個標的，與您在自選清單、K 線圖或下單面板當前選取的商品無關。因此它沒有連動（跟隨）與鎖定的切換，也沒有彈出成獨立視窗的按鈕——因為它不是格線裡的面板，而是頂部列上的常駐元件。

**資料更新與隱藏行為：** 徽章的數字由 10 秒一次的快照輪詢配合即時串流持續刷新。若當下取不到加權指數的價格（例如非交易時段或資料尚未就緒），整條狀態列會自動隱藏而不顯示空值；基差徽章也只有在台指期與加權指數同時有價、能算出價差時才會出現，缺一則暫時不顯示。

### 行情回放 Replay

在 Shioaji Pro 交易終端中，「行情回放（Replay）」功能是專為盤後檢討與練盤感而設計的。它透過 Shioaji API 的 `fetchHistoryTicks` 抓取當日的歷史成交 Tick，在面板內一張獨立、輕量化的折線圖（lightweight-charts 的 LineSeries）上，依時間逐筆重建盤中的價格走勢。您可以用它把一整天的行情「重播」一遍，反覆觀察某個轉折是怎麼形成的，而不必等到隔天開盤。這個面板由 ReplayPanel 元件負責，跟終端裡其他 K 線/五檔面板一樣，掛在 React Grid Layout 的工作區裡，並套用 PanelChrome 的標題列控制。

以下為您說明如何開啟與操作這個功能：

1. **新增面板：** 在交易終端頂部導覽列（HudHeader）右側，點擊「＋ 新增面板」選單按鈕，於彈出的「新增面板 Add Panel」清單中選擇「行情回放」，系統即會在您目前的工作區底部新增一個回放面板。此面板可開多個（非單例），例如同時比對兩檔商品的當日走勢。

2. **切換商品（連動）：** 面板預設為「連動」狀態，會跟隨您目前在終端裡選取的商品（透過自選清單、⌘K 搜尋、排行榜等任一路徑選取）。切換商品時，面板會自動重新抓取該商品的當日歷史 Tick 並重建折線圖。

3. **鎖定特定商品：** 若您希望這個回放面板固定盯著某一檔、不隨全域選取起舞，點一下標題列上的「連動」鈕（Link2 圖示）即可鎖定目前商品；鎖定後標題列會出現代碼輸入框，可直接輸入代碼（Enter 套用）。想恢復連動，再點一下「鎖定」鈕（Pin 圖示）即可。

4. **播放與暫停：** 面板左上角是「播放／暫停」按鈕。資料載入完成前按鈕會是停用狀態，載入好後點「▶ 播放」即開始逐筆重播，價格線會隨時間往右延伸；播放中按鈕變成「⏸ 暫停」，點一下可隨時停下。若已經播到最後再按播放，系統會自動跳回起點從頭重播。

5. **調整播放速度：** 播放按鈕右側有四段變速鈕「1x／5x／20x／100x」，目前生效的那一段會高亮顯示（預設為 5x）。愈高倍速，單位時間內餵入的 Tick 愈多，適合快速掃過整段行情；想細看某個瞬間就切回 1x 慢慢看。變速可在播放中即時切換。

6. **拖曳進度條定位：** 變速鈕右側是一條進度滑桿，範圍從第 0 筆到當日最後一筆 Tick。您可以拖曳滑桿快速跳到當天任一時點（例如直接拉到尾盤），折線圖會立即重繪到該位置，方便您反覆檢視同一段走勢。

7. **看狀態列：** 進度條右方的狀態文字會顯示目前重播到的價格與進度，格式為「價格 · 目前筆數／總筆數」；資料載入中會顯示「載入中…」，資料就緒但尚未播放時則顯示「首筆價格 · 0／總筆數」（例如「17250 · 0/48213」）。

8. **彈出獨立視窗：** 「行情回放」支援彈出（屬於 POPOUT_TYPES）。若您是多螢幕環境，點面板標題列右上角的彈出鈕（ExternalLink 圖示），即可把回放面板轉成獨立視窗放到第二螢幕，一邊看主畫面行情、一邊專心重播檢討。

**關於資料範圍與限制：** 本面板重播的是「歷史成交 Tick」而非即時行情——它是拿當日（期貨、選擇權會多抓前一交易日）已成交的紀錄來重建走勢，用於盤後與練盤，並不會即時跟盤更新。若該商品當天沒有可用的歷史成交（例如指數類商品沒有逐筆 Tick、或當日尚無成交），面板會直接顯示「無可回放的歷史成交」而不繪圖。此外，回放面板純粹是檢視工具，不涉及任何下單動作。

## 下單交易

### 下單面板 Order Ticket

在 Shioaji Pro 交易終端中，「下單面板（Order Ticket）」是您實際送出委託的地方。它針對「快、準、不誤觸」而設計：一份表單同時吃得下整股與零股、期貨與選擇權，會依商品自動切換欄位（股票給您「整股／零股」、期貨給您「新倉／平倉／當沖」倉別）。價格透過 Shioaji 的即時報價串流（useQuote）自動帶入牌告價，真正送單則是走 placeStockOrder／placeFuturesOrder。為了避免手殘，送出採「兩段式確認」，且每一筆在送出前都會先過一次風控檢查（checkOrderAllowed）。面板本身掛在 PanelChrome 標題列下，跟終端其他面板一樣支援連動與鎖定。

以下為您說明如何開啟與操作這個功能：

1. 如何開啟「下單面板」

**新增面板：** 在交易終端頂部導覽列（HudHeader）右側點「＋ 新增面板」選單，於「新增面板 Add Panel」清單中選擇 **「下單面板」**，系統即在您目前的工作區（React Grid Layout）版面最底部新增一個下單面板。

**跟隨或鎖定商品：** 下單面板可鎖定（pinnable）。預設為「連動」狀態（標題列顯示連動鈕），會跟著您在自選清單、⌘K 搜尋、排行榜、T 字或熱圖選到的商品走；若想固定只對某一檔下單，點標題列的連動鈕即可把它鎖定成目前商品，之後不再被全域選取影響。要恢復跟隨，點鎖定鈕回到連動即可。

**彈出獨立視窗（可選）：** 下單面板屬於可彈出面板，點標題列右上的彈出鈕，即可把它開成獨立視窗放到第二螢幕，一邊看圖一邊專心下單。

2. 選擇買進或賣出

**方向分頁：** 面板頂端有「買進 Buy」「賣出 Sell」兩個分頁，點一下即切換方向。您也可以用鍵盤熱鍵 **B（買進）／S（賣出）** 快速切換。每次切換方向都會自動解除已進入的確認狀態，避免帶著上一個方向的確認誤送。

3. 填入價格與數量

**價格：** 中間的價格欄位在您尚未手動編輯前，會自動帶入即時報價的最新成交價；您也可以用左右的「−／＋」鈕微調，或直接輸入。此外，若您在同一檔商品的 K 線圖上點價、或在五檔上點價，價格會被帶進這裡。一旦您動過價格，系統就不再用即時報價覆蓋您的輸入。

**數量：** 下方數量欄同樣有「−／＋」鈕與輸入框，數量單位會依商品自動顯示——期貨是「口」、股票整股是「張」、零股是「股」。

4. 設定價別與效期

**價別：** 股票提供 **LMT（限價）／MKT（市價）**，期貨另多一個 **MKP**。選到限價 LMT 時要填有效價格；切到市價類（MKT／MKP）時價格欄會鎖住並顯示「MKT」。提醒您：一旦選了市價類，系統會自動把效期切成 IOC；切回 LMT 則自動回到 ROD。

**效期：** 提供 **ROD（當日有效）／IOC（立即成交否則取消）／FOK（全部成交否則取消）** 三段，點選即套用。

5. 整股／零股，或期貨倉別

**股票—單位：** 下單股票時可切「整股」或「零股（盤中零股）」；選零股後數量單位會變成「股」。

**期貨—倉別：** 下單期貨或選擇權時，這一列變成倉別選擇：**自動 Auto／新倉 New／平倉 Cover／當沖 DayTrade**，供您指定這筆是開倉、平倉還是當沖。

6. 現股當沖先賣（沖賣）

**條件式出現：** 當您在「股票、賣出、整股」且該商品允許現股當沖（contract.day_trade 為可）時，會多出一個「沖賣」列。點開後即以「現股當沖先賣」送出（無券先賣，當日需回補）。這是先賣後買的當沖操作，請確認自己清楚當日必須回補再使用。

7. 停損停利保護（括號單）

**成交後自動掛保護：** 打開「停損停利保護」開關後，會出現停損價／停利價兩個輸入框。當您這筆進場單成交後，面板會自動為它登記一組 OCO 停損／停利保護單。請注意：這類保護是由 App 端負責監控觸發的，**只在交易終端開啟時有效，關掉程式即停止監控**，長時間離場請自行留意部位。

8. 送出前先看預估成本

**下單預估成本：** 送出鈕上方會即時試算這筆的成本概況——股票顯示成交金額、預估手續費，賣出時再加證交稅（一般股票當沖賣出減半估算）；期貨顯示契約值、乘數與每邊期交稅；選擇權顯示權利金與每邊期交稅。數字以牌告價估算，僅供參考。

9. 兩段式確認送出

**先預備、再確認：** 底部的下單鈕採兩段式。第一次點「買進下單／賣出下單」不會立即送出，而是進入「確認」狀態，鈕上會清楚列出「確認買進／賣出 數量 單位 @ 價格」讓您核對；**再點一次**才真正送出委託。切換買賣方向、或在價格框「輸入」新價格，會自動退出確認狀態並要您重新核對；但用 −／＋ 微調價格、或調整數量（±鈕或直接輸入），並不會退出確認，送出前請務必自行再次核對數量與價格。送出成功後下方會顯示委託回報，失敗則顯示錯誤原因。

10. 風控攔截與行情連線

**送單前先過風控：** 每一次真正送出前，系統都會先做風控檢查（對應頂部「風控」選單的 Kill Switch、單筆上限、日虧上限等規則）。若目前處於鎖定或超出上限，這筆會被攔下、不會送出，並在面板顯示被擋原因。**行情未連線時**下單鈕會顯示「⚠ 行情未連線，暫停下單」並無法點按，待連線恢復後才可下單。

**指數商品不可下單：** 若目前商品是指數（IND），面板只會顯示「指數商品僅提供即時行情與分析，不支援下單」——指數用於看盤與分析，本就無法委託。

### 閃電下單 Flash Order

在 Shioaji Pro 交易終端中，「閃電下單（Flash Order）」是一張為短線、當沖與搶帽子而生的價格梯（DOM trader）。它把 Shioaji 串流回來的即時報價與五檔委買委賣，攤成一條以「檔位（tick）」為單位、垂直排列的價格階梯：中間是價格欄，左側是買量、右側是賣量，讓您用「點哪一格就在哪個價位掛單」的方式下單，省去在傳統下單匣裡逐欄填價、填量、按送出的來回。技術上它訂閱 useQuote / useTradingLive 兩條串流即時重繪，梯身是一個固定視窗、隨現價在 tick 空間自動平移的 FlashOrder 元件，真正的委託則透過 placeQuickOrder 送進 Shioaji API。

以下為您說明如何開啟與操作這個功能：

1. **新增面板：** 在交易終端頂部導覽列（HudHeader）右側點「＋ 新增面板」，於彈出的「新增面板 Add Panel」清單中選擇「閃電下單」，系統就會在您目前的工作區（React Grid Layout）底部新增一張閃電梯。閃電下單不是單例，您可以同時開好幾張、分別盯不同商品。

2. **連動或鎖定商品：** 這張面板預設是「連動」狀態，會跟隨全終端目前選取的商品（由自選清單、⌘K 搜尋、排行榜等處決定）。若您想讓某一張梯固定盯住某檔，先點面板標題列（PanelChrome）上的「連動」鈕，即可把這張梯鎖定到目前全終端選取的商品；鎖定後標題列才會出現鎖定代碼輸入框，此時再輸入您想盯的代碼並按 Enter，就能把梯切換到該商品。（若只是想固定當前商品，點一下「連動」鈕即可。）再點一下鎖定鈕即可恢復連動。請注意：指數（IND）不可下單，切到指數時面板會顯示不支援下單的說明而非梯身。

3. **彈出獨立視窗（可選）：** 閃電下單支援彈出。點面板右上角的彈出鈕，即可把這張梯轉成獨立視窗放到第二螢幕，專心盯盤點價。

4. **⚡ 全開多面板：** 若您想一次擺出一整排閃電梯，當自選中含有可下單商品時，HudHeader 上會出現「⚡ 全開」選單。選好排版縮圖（8／6／4／3 檔的長條或右側直欄）後，系統會把自選前 N 檔各自開成一個獨立的閃電下單視窗，非常適合同時監控與操作多檔。

5. **設定下單數量：** 面板頂端的控制列有「量」欄位，點「＋／−」增減，或直接在數字框輸入張數（口數）。之後每一次點價、市價、平倉都以這個數量送出。

6. **先解除安全鎖（預設鎖定）：** 這是全面板最重要的安全設計——閃電下單預設是「鎖定」的。控制列上那顆按鈕在鎖定時顯示「啟用閃電下單」，點一下才會亮成「⚡ 點價即下單」進入啟用狀態；此時梯上的買賣格與市價鍵才會真正生效。未連線（行情非 LIVE）時按鈕會變成「⚠ 未連線」且無法啟用；只要行情連線中斷，面板會立刻自動退回鎖定，避免點進一條死掉的連線。任何時候按下 Esc 也會馬上重新上鎖。

7. **點價即下單（左買右賣）：** 啟用後，點某一價位左側的「買量」欄＝在該價位掛限價買單；點右側的「賣量」欄＝掛限價賣單。這就是「左買右賣、點價成單」的核心操作。鎖定狀態下這些格子是暗的、點了不會送單，游標移上去會提示「先啟用閃電下單」。

8. **現價自動置中與跟隨／固定：** 梯身會讓最新成交價自動保持在畫面中央（自動置中）；當現價逼近視窗上下緣時會自動重新置中，但只要您的游標停在梯身上就會暫停平移，確保價格不會在您點擊的瞬間跳動。控制列的「跟隨／固定」鈕可切換是否跟著現價跑，「置中」鈕則一鍵把現價拉回中央並恢復跟隨。您也可以用滾輪上下捲動梯身（會自動切成「固定」）、或在梯身上雙擊快速置中。若現價已捲出視窗，梯身邊緣會出現「▲／▼ 現價 xxx」的提示鈕，點它即可跳回現價。

9. **Order chip 一鍵刪單：** 當您在某價位有未成交的委託，該列的委託欄會出現一顆顯示剩餘口數的「委託 chip」（買單在左、賣單在右）。直接點這顆 chip，就會把該價位、該方向的所有在途委託送出刪單，不必再回到委託清單裡逐筆取消。

10. **市價買／賣、平倉、全刪：** 控制列下方的動作列提供幾顆快速鍵——「市價買」「市價賣」以市價立即送單（同樣需先解鎖；市價單會即時成交，請留意）；當本商品有淨部位時會多出一顆「平倉」鈕，一鍵以市價把整個部位反向平掉；最右邊的「全刪」會撤掉這檔商品目前所有未成交委託，按鈕上還會標出目前在途委託的筆數，沒有委託時呈灰色不可點。

11. **讀懂梯上的資訊：** 除了買賣量條，梯面還疊了不少即時線索——最新成交價那一列會附上成交量（×量）；您今日在某價位的成交會以小徽章標在委託欄外側；有部位時面板上方會顯示「多／空 口數 @ 均價」與該商品損益，均價所在的價位列也會被標記出來；漲停、跌停價位會分別以上下限色帶標示；梯底則以「Σ買／Σ賣」列出五檔委買、委賣總量。面板最底部一行提示會隨狀態切換：啟用時顯示「點買量＝限價買 · 點賣量＝限價賣 · 點單量＝刪單 · Esc 鎖定」，鎖定時顯示「安全鎖定中 — 點『啟用閃電下單』解鎖 · 滾輪捲動 · 雙擊置中」。

12. **安全與風控提醒：** 閃電下單送出的是真實委託，面板本身以「預設鎖定＋斷線自動上鎖＋Esc 隨手鎖＋防手殘重複點擊」四重保護把關；此外，若您在頂部「風控」選單啟用了 Kill Switch（風控鎖定），一般下單會被攔下，閃電梯自然也送不出委託——這是刻意的封鎖，並非故障。金額若看不清，可能是您在「主題」選單開了金額遮蔽，關掉即可還原損益顯示。

### 鋪單面板 Grid Ticket

在 Shioaji Pro 交易終端中，「鋪單（Grid Ticket）」面板是專為掛單交易者設計的——當您想在現價的上方或下方，一次沿著多個檔位鋪一整排限價單（例如網格接單、分批進場、掛牆），與其手動一筆一筆送，不如在這裡設好參數一鍵完成。它透過 Shioaji API 送出 ROD 限價委託，並依商品的跳動單位（tick size）逐檔換算出正確價格，同時遇到漲跌停會自動停下。所有由本面板送出的委託都會被打上 `custom_field = sjgrid` 標記，因此撤單與「動態跟隨」校正時，只會動到自己鋪的單，不會誤刪您在別處手動下的委託。

以下為您說明如何開啟與操作這個功能：

**新增面板：** 在交易終端頂部導覽列（HudHeader）右側點擊「＋ 新增面板」選單，於「新增面板 Add Panel」清單中選擇「鋪單」，系統即會在您目前的工作區（React Grid Layout）底部新增一個鋪單面板。此面板不是單例，您可以同時開多個、分別鋪不同商品或不同買賣邊。（提醒：鋪單面板不支援彈出獨立視窗，標題列不會出現彈出鈕。）

**跟隨或鎖定商品：** 本面板預設「連動」，會跟隨您在全終端目前選取的商品（透過自選清單、⌘K 搜尋、排行榜等任一路徑選取）。若想固定鋪某一檔，點標題列的連動鈕即可鎖定目前商品，或在鎖定輸入框直接輸入代碼；再點一下即恢復連動。請注意，指數（IND）不可下單，切到指數時面板會顯示不支援下單的說明而非鋪單介面。

**選擇買賣邊：** 面板上方有「買進鋪單／賣出鋪單」兩個分頁。選「買進鋪單」時，各檔限價會鋪在現價「下方」；選「賣出鋪單」時鋪在現價「上方」。切換買賣邊時，系統會自動關閉「動態跟隨」，避免舊方向的單被誤補。

**設定鋪單參數：** 四個帶「−／＋」按鈕的欄位可調整這排單的形狀——「起始檔距」（離現價幾個 tick 開始，1～50）、「檔數」（總共鋪幾檔，1～15）、「間隔(檔)」（每檔之間隔幾個 tick，1～10）、「每檔量」（每個檔位掛幾口/張，1～99）。下方的「預覽」列會即時顯示這排單的最外緣到最內緣價格區間與總檔數。實際算價會依該商品的跳動單位逐檔計算，若某檔已超出漲停或跌停價，就會在該檔停止、不再往外鋪。

**解鎖鋪單（安全鎖）：** 這是防誤觸的安全鎖。面板預設為鎖定狀態，您必須先點「解鎖鋪單」使其變為「⚡ 已解鎖」，下方的執行按鈕與「動態跟隨」才會生效。再次點擊會重新上鎖，並一併關閉動態跟隨。切換商品時，解鎖與跟隨狀態都會自動重置，需重新解鎖。

**一鍵靜態鋪單：** 解鎖後，點下方左側的「鋪 N 檔」按鈕，即會依現價一次送出這排限價單（每檔為 ROD 限價；期貨自動判斷開平倉，股票為整股）。此按鈕在「未連線」時會顯示「⚠ 未連線」並停用，送單過程中顯示「處理中…」。送出前會先過風控檢查（以「每檔量 × 檔數」估算總量），若超出風控上限會跳出「風控阻擋」而不送單；完成後會通知成功筆數（例如「買邊 5/5 筆」）。

**動態追價（動態跟隨現價）：** 若您希望這排單隨行情移動、始終維持與現價的相對距離，解鎖後可點「動態跟隨現價」。開啟後，系統每 2.5 秒校正一次：以「當前現價」重算理想的整排檔位，撤掉已跑掉的舊單、補上缺少的新單；為避免行情劇烈時瞬間爆量，每輪校正最多只做 4 個撤／補動作。請特別留意：這個追價校正是在 App 用戶端執行的（面板內的定時迴圈），只有在程式與面板開啟時才會運作，一旦關閉程式，自動撤補即停止，已在市場上的委託會維持原樣不再被照顧。

**一鍵全撤：** 右側的「全撤」按鈕會顯示目前在途的鋪單筆數，點下即批次刪除本商品、由本面板鋪出的所有委託（依 `sjgrid` 標記辨識，不會動到您的其他手動單）。執行全撤也會同時關閉動態跟隨。面板下方的「在途鋪單」列會列出目前掛在市場上的各檔買／賣價，方便您隨時核對。

**風險提醒：** 鋪單會「一次送出多筆委託」，動態跟隨還會自動撤補，屬於較進階、也較容易累積部位的操作。面板底部固定顯示風險提示，請務必先在「風控」選單確認單筆上限與日虧上限，並自行承擔下單風險。若已啟用風控鎖定（Kill Switch），一般鋪單會被攔截。

### 組合單 Combo Ticket

在 Shioaji Pro 交易終端中，「組合單 Combo Ticket」是為期貨轉倉、選擇權價差／跨式等「兩腳同時進場」策略設計的專用下單面板。它透過 Shioaji 的組合單 API（placeComboOrder）把兩張合約綁成同一筆委託送進交易所，並即時從每一腳的五檔委託簿合成出整組的買賣價（合成報價）——買進組合時付各買腳的賣價、收各賣腳的買價，讓您一眼看到整組策略當下的進出成本。面板還會依兩腳的商品、交割月份、履約價、Call/Put 與買賣別，自動判斷 TAIFEX 的組合類型（combo_type，如跨月價差、垂直價差、跨式、勒式），補上交易所有時無法自動推導的策略別，減少退單。

以下為您說明如何開啟與操作這個功能：

**1. 如何開啟「組合單」面板**

**新增面板：** 在交易終端頂部列（HudHeader）右側點「＋ 新增面板」，於彈出的「新增面板 Add Panel」清單中選擇「組合單」，系統即會在您目前工作區的最底部長出這個面板。

**單例面板：** 組合單是單例（singleton），整個工作區只會存在一個。若您已經開過，選單中的「組合單」會變灰並標註「（已存在）」，點不動。

**自己選腳、不跟隨全域商品：** 與 K 線圖、下單面板那類面板不同，組合單的標題列沒有連動／鎖定切換鈕，也不會跟著您在自選清單點選的商品變動——兩腳的合約完全由您在面板內自行輸入。它也不支援彈出為獨立視窗。

**2. 設定兩腳合約（腳 1／腳 2）**

**選買賣別：** 每一腳左側都有「買／賣」切換鈕；面板預設腳 1 為「買」、腳 2 為「賣」。

**輸入代碼：** 在代碼欄輸入合約（例如台指期 TXFF6、台指選 TX417000C6），按 Enter 或點到別處（失焦）即會解析。組合單只接受期貨與選擇權合約，輸入其他類型會將欄位標紅提示。

**看單腳報價：** 成功解析後，該腳下方會顯示合約名稱與即時「買 x／賣 y」，並標註這一腳是「付賣價」（買腳）或「收買價」（賣腳），方便您逐腳估算成本。

**3. 用「連動 T 字」快速帶入選腳**

**開啟連動：** 點面板上方的「連動 T 字」鈕（Link2 圖示）。開啟後，只要在「選擇權 T 字」面板點某個履約價的報價，系統就會自動把它填入「下一個未鎖定的腳」，並在兩腳之間交替填入；因此請搭配「選擇權 T 字」面板一起使用。

**鎖定某一腳：** 連動開啟時，每一腳會多出一個鎖頭鈕（Lock／Unlock）。把已經選好的腳鎖起來，T 字點擊就不會覆寫它，讓您專心用 T 字挑另一腳。

**4. 看合成報價、確認自動判別的策略**

**合成報價：** 兩腳都解析成功後，面板會顯示整組的「合成報價｜買／賣｜中價」，數字是把兩腳的即時五檔依買賣方向加減算出來的淨價。

**自動判策略：** 系統會依兩腳條件自動判斷組合類型並隨單帶給交易所——例如同商品跨月一買一賣＝跨月價差（轉倉）、同 Call/Put 不同履約價一買一賣＝垂直價差、同履約價一 Call 一 Put＝跨式、不同履約價一 Call 一 Put＝勒式等。這是為了補上交易所有時「無法自動推導策略別」而退單的情況，您通常不需手動指定。

**5. 設定方向、淨價、數量與條件**

**買進／賣出組合：** 在「組合」列選「買進組合」或「賣出組合」；也可用鍵盤 B／S 熱鍵直接切換整組方向。

**淨價與量：** 「淨價」欄會自動帶入合成中價（在您尚未手動改動前），您可自行覆寫；右邊「量」填整組口數。

**委託條件：** 「條件」可選 IOC（立即成交否則取消）或 FOK（全部成交否則取消）；只有「兩腳都是期貨」時才會多出 ROD（掛單等候、可掛芭樂價）。修改任一欄位都會取消已進入的確認狀態。

**6. 兩段式確認送出**

**先按一次「下單」：** 組合單採兩段式確認防誤觸。第一次點大按鈕只會「武裝」，按鈕文字會變成「確認買進／賣出組合 {口數} @ {淨價}（LMT/{條件}）」。

**再按一次確認：** 核對無誤後再點一次，才真正以限價（LMT）、您所選的條件、開平倉自動判（Auto）送出。成功會跳出「🧩 組合單已送出」通知。

**送不出去的情況：** 若行情未連線，按鈕會顯示「⚠ 行情未連線，暫停下單」且無法送出；兩腳尚未都解析完成時，按鈕會顯示「先輸入兩腳合約代碼」。

**7. 到價監控：合成價到價自動送 IOC**

**設定目標：** 在「到價」欄輸入目標淨價後點「啟動監控」。買進組合時，當合成賣價跌到（≤）目標即觸發；賣出組合時，當合成買價漲到（≥）目標即觸發。

**自動送單與上限：** 觸發後會自動以 IOC 送出（不論您在「條件」選了什麼，到價一律用 IOC），並內建防呆——最多嘗試 3 次、每次間隔 5 秒；達 3 次會自動停止並通知。修改任一腳也會自動解除監控。

**只在程式開著時有效（重要）：** 這個到價監控是在 App 端（此面板）即時運算的，只有在程式執行中、且此面板存在時才會盯價與送單；一旦關閉程式或移除面板，監控即停止，不會在雲端替您掛著。此外 IOC 可能部分成交後又再次觸發，請務必盯緊成交回報，避免重複建立部位。

**8. 管理在途組合單**

**查看：** 面板下方的「在途組合單」會列出仍在委託中的組合（約每 10 秒更新一次），以 +／− 標示各腳買賣方向，並顯示口數、淨價與狀態。

**刪單：** 每一筆右側都有「刪單」鈕，點下即送出組合刪單並更新列表。

### 風控 Kill Switch 與括號單

在 Shioaji Pro 交易終端中，「風控 Kill Switch 與括號單」是專為守住資金與情緒紀律而設計的下單安全機制。它的核心是 `risk.ts` 裡的一道守門函式 `checkOrderAllowed()`：每一條下單路徑在真正送出前都會先問過它，只要您按下手動鎖、或觸及您預設的單筆／日虧上限，這一筆委託就會被當場攔下。控制面板不是一個獨立視窗，而是常駐在頂部導覽列（HudHeader）的「風控」選單（RiskMenu）；至於「括號單」則內建在「下單面板」（OrderTicket）裡，由 `bracket.ts` 與客戶端觸價引擎（trigger-engine）接手，在您進場成交後自動替您掛上一組互斥的停損／停利保護。

以下為您說明如何開啟與操作這個功能：

**開啟「風控」選單：** 在交易終端頂部的導覽列（HudHeader）右側，找到「風控」選單按鈕並點開。平時它顯示「風控」兩字；一旦您啟動了鎖單，按鈕本身會變成「🔒 風控鎖定」，讓您一眼就能從畫面頂端確認目前正處於封鎖狀態。

**一鍵鎖單（Kill Switch）：** 選單最上方是紅色的「鎖定下單 Kill Switch」按鈕。點一下即進入鎖定，`risk.locked` 被設為開啟，之後所有一般委託都會被 `checkOrderAllowed()` 擋下並回報「風控鎖啟動中 — 所有下單已封鎖」。要恢復交易，再點同一顆按鈕（此時文字已變為「解除鎖定（恢復下單）」）即可。這是情緒失控或行情異常時最快的一道剎車，設定會存進瀏覽器本機（localStorage），重開程式仍然記得。

**啟用風控規則與上限：** 「鎖定下單」下方是「風控規則 Rules」區。先點「啟用風控規則」把總開關打開（開啟後顯示「✓ 規則啟用中」），再於底下兩個輸入框填入數字：**單筆上限** 限制每一筆委託的最大數量，超過即擋下並提示「超過單筆上限」；**日虧上限**（單位為新台幣）則在當日損益跌到負的該金額時封鎖後續下單。兩個欄位留白（顯示「不限」）代表不設限。請注意：這兩道上限只有在「規則啟用中」時才會生效，而 Kill Switch 手動鎖則不管規則開關、一律封鎖。

**確認當日損益：** 選單底部會顯示「目前當日損益估算」，數字由 App 以「持倉未實現＋期貨平倉損益」計算後餵給風控引擎（reportDailyPnl），方便您在設定日虧上限時對照目前水位。這裡也標明了一句重點：停損／停利觸價單不受風控封鎖。

**被攔截時會怎樣：** 當您在下單面板送單卻踩到上述任一條規則，委託不會送出，面板會直接以紅色「✕」回饋顯示被擋的原因（例如「當日虧損 … 已達上限，下單封鎖」）。這是一道純本機的攔截，發生在向後端送單之前。

**Esc 連按兩次全刪單：** Kill Switch 管的是「不讓新單出去」，若您要清掉「已經掛在市場上的未成交委託」，請在 0.6 秒內連按兩次 Esc，系統會執行 `cancelAllOrders()` 撤銷所有未成交委託（第一次按只會提示「再按一次 Esc 全部刪單」，避免誤觸）。兩者用途不同、可搭配使用：先 Kill Switch 止血，再 Esc×2 清場。

**開啟括號單（下單面板內）：** 括號單不是獨立面板，而是「下單面板」裡的一個選項。若您工作區還沒有下單面板，請先點頂部「＋ 新增面板」選單，從清單中選「下單面板」新增。面板中找到「括號單」那一列，點「停損停利保護」把它切為開啟（顯示「✓ 成交後自動掛保護」），下方隨即出現「停損價」與「停利價」兩個輸入框，填您要的價位（可只填其中一個）。

**送出進場單（兩段式確認）：** 下單面板採兩段式安全確認——第一次點「買進下單／賣出下單」只是「上膛」，按鈕會轉為「確認買進 … @ …」，需要您再點一次才真正送出；若行情未連線，按鈕會顯示「⚠ 行情未連線，暫停下單」。送出的進場單一樣要先通過上面的 Kill Switch／上限檢查。

**成交後自動掛 OCO 保護：** 進場單送出且您開了括號單後，系統先進入「🧷 括號單待命」。它透過兩條路徑偵測成交——即時的委託成交事件（SSE order_event，快速路徑）加上每 4 秒一次的委託查詢後援。一旦偵測到進場成交，`bracket.ts` 會自動掛出一組方向相反、彼此互斥（OCO）的觸價單：停損與停利各一張，成交後通知「🧷 括號單已啟動」。若進場單最後是被取消／失敗／未成交（Cancelled／Failed／Inactive），系統會放棄掛保護單並通知「保護單不掛」，不會留下孤兒單。當停損與停利其中一邊觸發，另一邊會自動撤銷並通知「OCO 互斥撤銷」。

**務必了解的客戶端限制：** 括號單的待命偵測與觸價單的執行都由本機的客戶端引擎負責——觸價條件是靠監看即時 Tick 串流（SSE）判斷、達價後才送出市價單。這代表它「只在 App 開啟時監控，關掉程式即失效」；雖然待命與觸價設定會存在本機，但只要終端沒開著，就不會有任何一張保護單被觸發。這些保護性的觸價出場單設計上會略過 Kill Switch（bypassRisk），確保您鎖單止血時，既有部位的停損／停利仍能正常出場。

**連動與彈出：** 下單面板為可連動面板，預設跟隨您在自選清單／⌘K／排行等處選取的商品，也可用標題列的「連動／鎖定」鈕把它釘死在特定代碼；若您用多螢幕，點面板右上角的彈出鈕即可把下單面板轉為獨立視窗，專心盯一檔操作。至於「風控」選單則固定常駐在頂部導覽列，隨時可從畫面最上方一鍵封鎖或解除。

## 選股分析

### 排行榜 Scanner

在 Shioaji Pro 交易終端中，「排行榜 Scanner」是為盤中快速選股、抓強勢股與弱勢股而設計的。它透過 Shioaji API 的掃描器排行（fetchScanner）抓取全市場即時排行資料，在 ScannerPanel 面板上以漲幅、跌幅、成交量、成交額等不同維度，為您列出當下最值得注意的標的，並每 15 秒自動更新一次。它不只是單維排行，還內建一組「複選」的量價交叉篩選，讓您用多條件同時過濾，甚至反向篩出放空候選。

以下為您說明如何開啟與操作這個功能：

**新增面板（單例）：** 在交易終端頂部導覽列（HudHeader）右側點擊「＋ 新增面板」選單，於彈出的「新增面板 Add Panel」清單中選擇「排行榜」，系統即會在您目前的工作區（React Grid Layout）中放入排行榜面板。排行榜屬於單例面板，且預設版面的左欄通常已內建一個；若工作區已存在同型面板，該選項會顯示「（已存在）」且點不動，代表您直接使用現有那一個即可。此面板不支援彈出為獨立視窗。

**切換排行類別：** 面板頂部有一排切換鈕——「漲幅」「跌幅」「量」「額」「複選」。點任一個即切換排行維度（漲幅／跌幅走 ChangePercentRank、量走 VolumeRank、額走 AmountRank），單一維度模式預設抓前 20 名。您所選的類別會被記住（存於瀏覽器 localStorage），下次開啟沿用上次的選擇。

**複選多條件篩選：** 點「複選」會在下方展開一列篩選條件，讓您同時套用三個門檻：漲幅≥%（預設 2）、量≥千張（預設 5）、額≥億（預設 1）。系統會把漲幅、量、額三張深度排行聯集後，交叉篩出「同時符合全部三個門檻」的標的（最多 30 檔）。這樣設計是為了避開漲幅榜最前段常見、實際難以交易的標的，改用量價交叉找出真正量價俱足、可操作的名單。三個數字都可直接在輸入框裡修改，改完會即時記住。

**放空篩選：** 在「複選」模式的篩選列左側有「做多／放空」切換鈕。切到「放空」後，原本的「漲幅≥%」條件會翻轉成「跌幅≥%」，改以跌幅＋量＋額交叉，並把跌最多的弱勢股排在前面，方便您篩選放空候選標的。這個做多／放空的選擇同樣會被記住。

**顯示類別與跳轉熱力圖：** 每一列在代碼、名稱之後，會標出該股所屬的類別（產業別）小標籤。這個標籤本身是一顆可點的按鈕（滑鼠移上去顯示「跳到此類股熱力圖」），點它會直接聚焦到「類股熱力圖」面板對應的類股。因為它會攔截點擊，所以單獨點類別標籤不會順帶選取該檔。

**點選標的、連動全終端：** 點任一列，該商品即被選取並連動整個交易終端——K 線圖、五檔、下單面板、成交明細等所有「連動」面板會一起跳到這檔，被點的那一列也會以高亮標示目前選取。排行榜本身是一個「選取來源」，它不跟隨也不能鎖定特定商品（標題列沒有連動／鎖定切換）。請注意：這個動作是「連動選取」，把全終端切到這檔商品，並不是把它加入自選清單。

**自動更新與重試：** 面板每 15 秒會自動重新抓取一次排行，盤中無需手動刷新。若排行資料一時取不到，面板會顯示「排行資料無法取得」訊息與一顆「重試」按鈕，點「重試」即可重新載入。

### 類股熱力圖

在 Shioaji Pro 交易終端中，「類股熱力圖（Sector Heatmap）」功能是專為由上而下的選股節奏而設計的——先一眼看出今天資金往哪些類股集中，再點進強勢類股裡挑個股。它以 Shioaji 的快照資料（fetchSnapshots）為底：總覽層抓各 TWSE 類股指數（IND）今日漲跌幅來上色，並把最強的類股自動排到最前；進到單一類股後，改抓該類股成員個股（STK）的快照，依成交額大小排列、用顏色深淺表示漲跌幅。類股指數還會透過即時報價串流（quote store）持續更新，所以盤中色塊會隨行情變動。整體是一個嵌在工作區（React Grid Layout）中的單例面板。

以下為您說明如何開啟與操作這個功能：

1. **如何開啟「類股熱力圖」面板：** 在交易終端頂部列（HudHeader）右側點「＋ 新增面板」選單，於彈出的「新增面板 Add Panel」清單中選擇「類股熱力圖」，系統就會在您目前的工作區底部新增一個熱力圖面板。此面板為單例（singleton），一個工作區只會有一個；若已經開著，選單上的該項目會變灰並標註「（已存在）」，點不動。您也可以直接套用內建的「熱力選股」預設版面（版面選單 → Presets），一次幫您把熱力圖擺好位置。

2. **第一層 類股總覽：** 面板開啟後預設停在「類股總覽」。這裡把各主要類股指數排成一格格色塊，顏色代表今日漲跌幅（漲跌方向會跟隨您在「主題」選單設定的漲跌顏色，紅漲綠跌或綠漲紅跌皆可；幅度在約 ±5% 時顏色最飽和），而且會把最強的類股自動排到最前面。每個色塊顯示類股名稱與漲跌百分比，工具列並提示「各類股指數漲跌 · 點一下進該類股」；滑鼠移上去會出現「XX指數（+X.XX%）」的說明。

3. **點進類股看個股：** 在總覽層點任一個類股色塊，面板就會切到第二層，展開該類股的成員個股。第二層的色塊改以個股當日漲跌幅上色，並依「成交額」由大到小排列（工具列右側標示「依成交額排序 · 色深=漲跌幅」），最多顯示 80 檔。每格顯示股票代碼、名稱與漲跌幅。

4. **切換類股：** 進入個股層後，工具列中間會出現一個類股下拉選單，直接展開就能改看別的類股（選單只列出成員達 5 檔以上的類股，並在每項後面標出檔數）。您上次瀏覽的類股會被記住（存在瀏覽器 localStorage），下次開面板會自動回到同一類。

5. **回到總覽：** 個股層工具列左上角有「← 總覽」按鈕，點一下即可退回第一層的類股總覽，重新綜覽全市場強弱。

6. **點個股連動全終端：** 這個面板是「選股來源」而非跟隨商品的面板——在個股層點任一檔股票的色塊，會把該商品連動（onPick）到整個終端，也就是您所有處於「連動」狀態的 K 線圖、五檔、下單面板、成交明細等都會同步切到這一檔。也因為它本身就是選取的來源（與自選清單、排行榜同一類），它的標題列沒有「連動 ↔ 鎖定」切換鈕，也不需要您指定要跟隨哪個商品。

7. **從排行榜「跳同類」跳進來：** 若您在排行榜點某檔的「跳同類」，終端會直接把熱力圖切到該股所屬的類股（第二層），方便您就地檢視整個族群當天的表現。

8. **資料更新與使用限制：** 兩層的色塊都以約 20 秒為週期輪詢 Shioaji 快照更新，類股指數另外吃即時報價串流，因此盤中顏色會持續變動；盤後或尚無資料時，總覽會顯示「類股指數載入中…」，個股層則顯示「此類股無資料」。此面板為單例，且不支援彈出成獨立視窗（標題列沒有彈出鈕），若要多螢幕檢視，建議搭配可彈出的 K 線圖、五檔等面板一起使用。

### 個股籌碼卡

在 Shioaji Pro 交易終端中，「個股籌碼卡（籌碼資訊）」面板是為選股與進場前的風險確認而設計的。它把一檔股票的融資融券配額、可借券源、處置警示、當沖資格等「籌碼面與交易限制」資訊集中成一張卡片，讓您在下單前就能一眼確認「這檔能不能當沖、有沒有被處置、還有沒有券可以空」。技術上，它由 ChipsCard 元件呈現，透過 usePoll 每 60 秒向 Shioaji 資料 API 輪詢三路資料——融資券配額查詢（credit_enquire）、借券源（short_stock_sources）、當日處置股清單（regulatory_punish）——再搭配合約本身的靜態欄位（當沖資格 day_trade、融資／融券餘額）組合成幾張統計小卡顯示。

以下為您說明如何開啟與操作這個功能：

**如何開啟「籌碼資訊」面板：** 在交易終端頂部的導覽列（HudHeader）右側，點擊「＋ 新增面板」選單按鈕（AddBlockMenu），在彈出的「新增面板 Add Panel」清單中選擇 「籌碼資訊」，系統即會在您目前的工作區（React Grid Layout）最底部新增一個籌碼卡面板。此面板不是單例，您可以同時開好幾個，分別鎖定不同的股票對照觀察。

**跟隨當前商品（連動）：** 面板預設為「連動」狀態，會自動跟隨您在自選清單、⌘K 商品搜尋、排行榜或熱力圖點選的全域商品。只要切換標的，籌碼卡就會即時換成該股的資料。

**鎖定特定股票：** 若您想讓某一張卡固定盯著特定股票，點一下標題列上的「連動」鈕（Link2 圖示），它會把目前商品鎖定；標題列改成顯示可編輯的代碼輸入框，您也可以直接輸入代碼（Enter 套用）。要恢復跟隨全域選取時，再點一下「鎖定」鈕（Pin 圖示）即回到連動。

**只支援股票商品：** 籌碼卡的資料僅對上市櫃個股（STK）有意義。若當前商品是期貨、選擇權或指數，面板會直接顯示「籌碼資訊僅支援股票商品」；資料尚在載入時則顯示「載入籌碼資訊…」。

**讀懂卡片內容：** 面板以數張統計小卡呈現，主要包含——「處置/警示」（是否在當日處置股清單中，被列為處置時顯示「⚠ 處置股」）、「當沖資格」（可當沖／僅可先買／不可當沖，取自合約的 day_trade）、「融資成數 / 餘額單位」與「融券成數 / 餘額單位」（來自融資券配額查詢）、合約上的「融資餘額」與「融券餘額」，以及有取到資料時的「可借券源」。其中會影響交易的警示項（處置股、非「可當沖」）會以漲色標色，讓您一眼就看到限制。這些卡片是唯讀資訊，面板內沒有需要您操作的按鈕或輸入欄位，純粹供您在下單前查核。

**資料更新頻率：** 卡片每 60 秒自動輪詢一次最新配額與處置狀態，您不需要手動刷新；切換或鎖定不同股票時也會立即重新抓取該檔資料。

**彈出為獨立視窗（可選）：** 籌碼資訊屬於可彈出的面板。若您使用多螢幕，可點擊面板標題列右上角的彈出鈕（ExternalLink 圖示），把這張籌碼卡轉成獨立視窗放到第二螢幕，一邊看盤一邊盯籌碼；籌碼卡不論在網格內或彈出視窗，皆固定每 60 秒輪詢一次，間隔不會因彈出而改變。若不再需要，點標題列的關閉鈕（X 圖示）即可移除此面板。

## 選擇權

### 選擇權 T 字報價與損益圖

在 Shioaji Pro 交易終端中，「選擇權 T 字報價」與「選擇權損益圖」是一組專為台指選擇權（TXO）操作而設計的搭檔面板。T 字報價把買權（CALL）、履約價、賣權（PUT）左右對排成一張表，讓您一眼掃過整排價外價內的成交與買賣報價；損益圖則把您手上的選擇權持倉（或您自己假設的模擬部位）攤成一條「到期結算損益曲線」，用來檢視這組部位在結算日不同點位下的盈虧輪廓。

技術上，T 字面板（OptionChain）只載入一次 TXO 合約清單並快取，報價以 `fetchSnapshots` 批次快照每 5 秒更新一次；它以台指期 TXFR1 的即時價作為價平（ATM）錨點，自動往上下各展開 8 檔履約價。損益圖面板（OptPayoff）則把您期貨帳戶裡的選擇權持倉，逐筆透過 `ensureContract` 解析成「腳（leg）」，在一張 canvas 上以純結算內含值畫出損益曲線——不含時間價值與手續費。兩者跨視窗連動則走 BroadcastChannel，讓彈出到第二螢幕的 T 字也能連動主視窗的下單面板與組合單。

以下為您說明如何開啟與操作這個功能：

**新增面板：** 在頂部導覽列（HudHeader）右側點「＋ 新增面板」選單，於清單中選「選擇權 T 字」；要看損益圖則另外再選「選擇權損益圖」。這兩個都是單例（singleton）面板，工作區各只能有一個，若已經開過，該選項會顯示為灰色並標註「（已存在）」，點不動。

**切換到期月份（T 字）：** T 字上方工具列會列出近月的到期月份按鈕（最多 6 個，取自合約的 delivery_month），點一下即切換到該月合約；您選的月份會記在瀏覽器（localStorage），下次開啟自動沿用。

**看價平錨點（T 字）：** 工具列右側顯示「TXF {價格}」，那是台指期 TXFR1 的即時價與漲跌；T 字就以它為中心展開履約價，最接近現價的那一格履約價會被特別標示，方便您快速定位價平。

**讀報價（T 字）：** 表格左半是「CALL 買權」、中間是「履約價」、右半是「PUT 賣權」，買賣權各自顯示成交／買／賣三欄；報價每 5 秒批次快照更新一次，並非逐筆即時跳動，屬於盤後或觀盤用的輕量刷新。

**點 T 字連動全終端、湊齊組合單兩腳：** 點某一列時，系統會依您點在該列的左半或右半判斷方向——點左半連動該履約價的「買權」、點右半連動「賣權」（面板提示：「點 CALL 側連動買權、PUT 側連動賣權」）。這一點同時做兩件事：一是把該選擇權設為全終端目前選取商品，讓下單面板、五檔、K 線等連動面板一起跟著切換；二是把這隻腳丟給「組合單」面板（若它處於連動模式）。因此，先點一個 CALL 側、再點一個 PUT 側，就能把組合單的兩腳依序帶進去。

**關於連動與鎖定（T 字）：** 選擇權 T 字本身就是一個「選取來源」，和自選清單、排行榜、熱力圖一樣是主動去連動別人；它的標題列沒有「連動↔鎖定」切換鈕，也不需要跟隨或鎖定某一檔商品。

**彈出獨立視窗（T 字）：** T 字面板支援彈出，標題列右上有彈出鈕，可把它丟到第二螢幕專心盯報價。彈出視窗裡點 T 字時，會透過 BroadcastChannel 通知主視窗切換選取商品，主視窗的下單面板等連動面板照樣跟著動，組合單也一併跨視窗連動。

**開啟損益圖並讀入持倉：** 「選擇權損益圖」同樣從「＋ 新增面板」加入。開啟後它會讀取您期貨帳戶中的選擇權持倉，逐筆解析成腳並畫出到期結算損益曲線；若目前沒有選擇權持倉，面板會提示「無選擇權持倉 — 可在下方加入模擬部位」。

**選到期月份、勾選要納入的腳（損益圖）：** 圖表左上的下拉選單可依到期月份篩選要納入計算的腳。下方腳位清單中，每一列前面都有勾選框，讓您把某一筆持倉暫時排除或重新納入，即時重畫曲線。

**加入買方／賣方模擬部位（損益圖）：** 面板底部一排欄位可組出模擬腳——先用「買／賣」切換決定方向（買方與賣方的到期損益曲線方向相反，就由這個切換決定正負），再選 Call／Put／期貨，填入履約價（選期貨時此欄自動隱藏）、口數、以及權利金（期貨則為成交價），最後按「＋模擬」加入。模擬腳會標上燒瓶圖示，可隨時按該列的 X 移除；真實持倉的腳則不可刪除，只能勾選納入與否。

**讀損益曲線（損益圖）：** 曲線畫在 canvas 上，獲利段以上漲色、虧損段以下跌色呈現，並附零軸、各履約價的虛線標記，以及台指期現價（TXFR1）的虛線標記，左上角另標出這組部位的最大／最小損益（max／min）。請留意工具列的警語：這是「結算損益試算，不含時間價值／手續費，僅供參考，盈虧自負」——它畫的是到期日的內含價值損益，不是目前市值的即時盈虧。

**損益圖不可彈出：** 與 T 字不同，選擇權損益圖不在可彈出面板之列，標題列沒有彈出鈕，只能留在主工作區的格線中使用。

## 帳務風控

### 持倉／委託／帳務／損益分析

在 Shioaji Pro 交易終端中，「持倉／委託／帳務／損益分析」 是整個帳戶的風控駕駛艙。它由兩個單例（singleton）面板協同組成：一個是 「持倉/委託/帳務」面板（原始碼中的 BottomDock），把您選定帳號的即時持倉、未成交委託與帳務水位收攏成三個分頁；另一個是 「損益分析」面板（PnlPanel），專責回顧近 30 日的已實現戰績。資料全部來自 Shioaji API——持倉與委託由後端 API 定期輪詢刷新（並非串流推送；useTradingLive 只是行情連線旗標，用來在行情未 LIVE 時停用平/反下單，並不負責更新持倉/委託資料），帳務水位取自帳戶餘額與期貨保證金，損益曲線則向 `/api/v1/portfolio/profit_loss` 分別撈取證券(S)與期貨(F)兩本帳。畫面上的資產分布甜甜圈以 canvas 繪製、權益曲線以 SVG 折線呈現。這兩個面板顯示的都是「全帳戶」資料，因此不跟隨您在自選清單點選的商品。

以下為您說明如何開啟與操作這個功能：

**新增「持倉/委託/帳務」面板：** 在頂部導覽列（HudHeader）右側點「＋ 新增面板」選單，於清單中選擇 「持倉/委託/帳務」。它是單例面板，預設版面（中欄）通常已內建一個；若工作區已存在，選單中該項會標「（已存在）」而點不動。此面板呈現全帳戶資料，與您選取的商品無關，所以標題列沒有連動/鎖定鈕，也沒有彈出鈕。

**切換要查詢的帳號：** 面板上緣是三個分頁——「持倉 Positions [持倉檔數]」「委託 Orders [有效/總數]」「帳務 Account」（「持倉」括號內是目前持倉的檔數／筆數，也就是有幾列持倉，不是合計口數或張數）。分頁列最右側有 證券(S) 與 期貨(F) 兩個帳號下拉，切換後所有分頁都會改查該帳號的資料（帳號號碼在隱私模式下會被遮蔽）。

**持倉分頁——即時損益一目了然：** 表格逐列列出代碼、名稱（自合約快取補上，不再只看代碼）、方向（多 LONG／空 SHORT）、數量、成本、現價、損益、報酬率，以及一條以中線為零點、獲利向右／虧損向左的「損益分布」長條（長條顏色綁定「主題」選單的「漲跌顏色」設定：台股預設「紅漲綠跌」時，獲利顯示為紅、虧損顯示為綠；若要以綠代表獲利，需切換為「綠漲紅跌」）。股票數量以「張/股」顯示、期貨以「口」顯示。點任一列會連動圖表與下單面板（跳到該商品）。

**一鍵平倉／反手（附安全鎖，請務必留意）：** 每列右側有 「平」 與 「反」 兩顆按鈕——「平」是對整個倉位送出反向市價單沖銷，「反」則是反向兩倍口數（翻倉）。這是會真實成交的市價單，因此表頭預設有一個鎖頭鎖定它們（防止存股族誤觸一秒反向）；您必須先點表頭的鎖頭解鎖，按鈕才會啟用。此外，當行情串流未連線（顯示未 LIVE）時，平/反會自動停用暫停下單。股票的平/反會自動把股數拆成「整張(Common)＋零股(盤中零股)」混合下單，送出後跳出「平倉單已送出／反手單已送出」通知。

**委託分頁——刪單、改價、改量：** 委託列表以最新在上排序，顯示買賣、類別（價別/效期，期貨另標新倉/平倉/當沖、股票另標零股/定盤/鉅額）、委託量、成交（部分成交會有進度條）、狀態與回報訊息。仍在有效狀態（委託中/部分成交等）的單子，列尾提供 改價（限價單才可，輸入新價格）、改量（只能減量，輸入新數量）與 CANCEL 三個動作；改單成功會跳「改價已送出／改量已送出」通知。若您想一次撤掉所有未成交委託，也可用鍵盤在 0.6 秒內連按兩次 Esc（第一次會先提示「再按一次 Esc 全部刪單」）。

**帳務分頁——權益數、保證金與資產分布：** 這一頁把水位攤開成一張張卡片：證券交割帳戶餘額、股票市值（已扣估算的賣出手續費與證交稅，ETF 稅率另計）、資產市值（股票市值＋期貨權益＋交割餘額的加總）、期貨的權益數／可用／原始／維持保證金、風險指標（TAIFEX 口徑，低於 100% 標紅示警有追繳與代沖銷風險、低於 200% 標黃）、期貨平倉損益，以及近日交割款。下方還有一個 「資產分布」甜甜圈（證券／期貨權益／交割帳戶三色分佔，圓心顯示資產市值）與 「前五大持股」 長條，讓您一眼看出資金集中在哪。

**新增與彈出「損益分析」面板：** 一樣從「＋ 新增面板」選 「損益分析」（同為單例）。它有個特別之處：雖然不能鎖定商品，卻可以彈出成獨立視窗——點面板右上角的彈出鈕，就能把它丟到第二螢幕上專心檢討盤後績效。

**看懂損益分析的數字：** 面板頂部是 「30 日已實現損益」 大字（依總額正負變色），右側一條累積 「權益曲線」（把每筆損益逐日累加畫成折線，收在正值以「漲」色、負值以「跌」色呈現）。這裡的漲／跌配色同樣依「主題」選單的「漲跌顏色」設定：台股預設「紅漲綠跌」時，正值（獲利）為紅、負值為綠；切換為「綠漲紅跌」才會以綠代表正值。下方四張卡片分別是 筆數／勝率、平均獲利、平均虧損 與 賺賠比（平均獲利對平均虧損的倍數）。資料同時涵蓋證券與期貨兩本帳，每 60 秒自動更新一次；若近 30 日沒有已實現損益或資料撈取失敗，會顯示對應提示。

**搭配隱私與風控：** 若要截圖分享，可到頂部「主題」選單底部開啟「金額遮蔽」，本功能的數量、損益、權益、水位等金額都會被遮成圓點（唯獨風險指標百分比不遮）。另外，頂部「風控」選單的 Kill Switch 可鎖定一般下單作為總開關；請注意這類客戶端層級的保護（含平/反的安全鎖）都只在 App 開啟時有效，關掉程式即失效，不會在後端替您持續看守。

## 介面系統

### 自訂版面與多視窗

在 Shioaji Pro 交易終端裡，整個工作區並不是一張寫死的固定畫面，而是一面可以自由拼貼的「面板牆」。每一塊功能——K 線圖、五檔、下單面板、成交明細、排行榜——都是一個獨立的面板（Block），漂浮在底層的 React Grid Layout 格線上。這個「自訂版面與多視窗」機制，就是為了讓不同交易風格的人（做當沖的、盯選擇權的、鋪單掛量的）都能把最順手的工具擺到自己習慣的位置，而不必遷就別人的版型。所有面板的搬移、縮放、新增、連動/鎖定與版面存檔，都由 `workspace.ts` 統一管理，並即時寫回瀏覽器的 localStorage（目前工作區存於 `sj-pro-workspace-v2`，您自己命名的版面存於 `sj-pro-profiles-v1`），下次開啟自動還原；每塊面板頂端的標題列，則由共用的 PanelChrome 元件負責，一致地提供拖曳、連動/鎖定切換、彈出與關閉等控制。

以下為您說明如何開啟與操作這個功能：

**新增面板：** 在交易終端頂部導覽列（HudHeader）右側，點擊「＋ 新增面板」選單。點開後會彈出標題為「新增面板 Add Panel」的下拉，裡面依序列出全部 24 種面板（K 線圖、五檔、下單面板、成交明細、閃電下單、籌碼資訊、分價量表、行情回放、委託簿熱圖、鋪單……）。點任一項，系統就會在您目前工作區的最底部新增一個該類型面板，並套用它預設的寬高與最小尺寸。這是新增面板的唯一入口——沒有拖曳新增，也沒有右鍵選單。

**留意單例面板：** 有些面板天生只該存在一個（例如自選清單、排行榜、持倉/委託/帳務、通知中心）。當工作區裡已經有這種面板時，選單中對應的項目會變成灰色不可點，並在名稱後加註「（已存在）」，避免重複開啟。

**多開同型面板：** 相對地，像 K 線圖、五檔、下單面板、成交明細、閃電下單、行情回放這類面板並非單例，您可以重複新增多個。想同時盯兩檔標的的走勢？就開兩張 K 線圖，一張連動、一張鎖定，並排對照。

**拖曳移動：** 每個面板的整條標題列就是拖曳把手。按住標題列往任意方向拖，面板就會在格線上移動，其他面板會自動讓位重排。標題列上的按鈕、輸入框與下拉選單都被特別排除在拖曳判定之外，所以您點連動鈕、打鎖定代碼時，不會不小心把整塊面板拖走。

**縮放大小：** 把游標移到面板的邊緣或右下角，即可拖動改變寬高。每種面板都有各自的最小尺寸限制（minW / minH），縮到下限就不會再變小，確保內容不會被擠壓到無法閱讀。

**連動或鎖定商品（每個面板各自獨立）：** 會吃商品的面板（K 線圖、五檔、下單、成交明細、閃電下單、籌碼、分價量表、行情回放、委託簿熱圖、鋪單）標題列右側會有一顆切換鈕。預設為「連動」狀態（鏈結圖示），面板會跟隨全終端目前選取的商品——也就是您在自選清單點選、或用 ⌘K 搜尋跳轉的那一檔。點一下「連動」鈕，面板就把當前商品「鎖定」下來（變成 Pin 圖示），之後不論全域怎麼切換，這塊面板都只顯示這一檔。鎖定狀態下標題列會出現一個代碼輸入框，您可以直接改輸代碼、按 Enter 套用（系統會自動去空白、轉大寫）；想恢復跟隨，再點一下「鎖定」鈕即可回到連動。至於自選清單、排行榜、選擇權 T 字、熱力圖這類「本身就是選商品來源」的面板，以及顯示全帳戶資料的損益/帳務面板，則沒有這個切換——它們要嘛負責決定全域商品，要嘛與個別商品無關。

**彈出獨立視窗（多螢幕）：** 部分面板的標題列上會有一顆「彈出」鈕（外連圖示）。點它會把該面板轉成一個獨立的作業系統視窗，方便您拖到第二螢幕專心盯盤。支援彈出的面板共十一種：K 線圖、五檔、下單、成交明細、閃電下單、籌碼、分價量表、行情回放、委託簿熱圖、選擇權 T 字與損益分析；沒有這顆鈕的面板就不支援彈出。（彈出視窗會採用較長且帶隨機間隔的輪詢頻率抓資料，避免多開視窗時同時把上游行情打爆。）

**移除面板：** 每個面板標題列最右邊的「✕」鈕，會把該面板從版面移除，工作區立即存檔。

**套用內建預設版面：** 若您懶得從零擺放，可在 HudHeader 的「版面」選單裡挑選內建的預設版型。目前提供多組常見情境的排版：標準看盤、當沖交易、雙圖對照（一張連動圖＋一張鎖定台指期 TXFR1 的圖並排）、選擇權、衍生商品探索、鋪單交易、閃電矩陣（自選＋四條閃電梯，第一條連動、其餘鎖定熱門檔）、熱力選股、AI 副駕、分析研究。滑鼠停在每個預設上會顯示它的用途說明，點選即整組套用、覆蓋目前版面。

**命名儲存與載入自己的版面：** 同樣在「版面」選單，您可以把目前這套辛苦擺好的版型輸入一個名稱存下來（同名會直接覆蓋）。存過的版面會列在選單中，隨時可一鍵載入或刪除。所有具名版面都只保存在本機瀏覽器，不會上傳。

**重設回預設：** 選單底部有「↺ 重設為預設版面」，一鍵回到出廠的標準七面板配置（自選＋排行 / K 線＋持倉 / 五檔＋下單＋明細）。

**自動保存，隨開隨用：** 您的每一次拖曳、縮放、新增、刪除、切換連動/鎖定，系統都會即時寫回本機。下次開啟終端會自動還原上次的版面；萬一存檔資料損毀而無法通過驗證，系統會安全地回退到預設版面，不會卡在壞畫面。

（提醒：「AI 副駕」版面用到的「AI Agent」面板，以及「策略回測」面板，屬於桌面版專屬的閉源功能模組；在開源 Web 版套用相關版面時，該面板會顯示鎖定畫面而非實際內容，敬請留意。）

### 商品搜尋（⌘K）與快捷鍵

在 Shioaji Pro 交易終端中，「商品搜尋（⌘K）」與一組全域交易快捷鍵，是為了讓您手不離鍵盤、快速切換商品與操作下單而設計的。它的底層是一個掛在整個視窗上的全域鍵盤監聽器（`use-hotkeys.ts` 的 keydown listener），負責攔截 ⌘K／Ctrl+K、B、S、以及連按兩次的 Esc；而按下 ⌘K 後彈出的搜尋框，則是獨立的 `CommandPalette` 元件，透過 `searchProducts` 即時查詢相符的商品，選定後以 `primeContract` 預熱合約快取、再呼叫跳轉。這一整套快捷操作不是某個面板，而是常駐在終端最上層的「介面系統」，因此不需要新增面板、也沒有彈出視窗，隨時按鍵就能用。

以下為您說明如何開啟與操作這個功能：

1. **開啟商品搜尋（按 ⌘K）：** 在終端任何位置按下 **Cmd+K（Mac）或 Ctrl+K（Windows）**，畫面上就會彈出置中的商品搜尋框。這個快捷鍵是「優先攔截」的——即使您當下正停在某個輸入框或下拉選單裡打字，⌘K 仍然有效（它會先攔下並阻止預設行為，再開啟搜尋框），所以不必先點空白處也能叫出它。

2. **輸入代碼或股名搜尋：** 搜尋框的提示文字是「輸入代碼跳轉商品（2330、TXFR1…）」。您可以輸入股票代碼（如 `2330`）、期貨代碼（如 `TXFR1`），也支援用中文股名搜尋；系統會在您停頓約 0.12 秒後即時查詢，最多列出 8 筆相符的建議，每一筆都會顯示「代碼、名稱、類別」三欄，方便您確認是不是要找的那一檔。

3. **選取並跳轉商品：** 有了建議清單後，用 **↑／↓ 方向鍵**上下移動選取（會循環到頭尾），按 **Enter** 就跳轉到目前選中的那一檔；也可以直接用**滑鼠移到某一列並點擊**選取。若您沒有等建議出現、直接按 Enter，系統會拿您輸入的文字（自動轉大寫）當代碼嘗試跳轉。查詢過程中會顯示「查詢中…」，若這個代碼查無商品，則會提示「找不到此商品代碼」，讓您重新輸入。

4. **跳轉即連動全終端：** ⌘K 的跳轉走的是全域選取路徑——選定商品後，它會設定終端目前的「全域選取商品」。因此所有處於「連動」狀態（未鎖定）的面板，例如 K 線圖、五檔、下單面板、成交明細等，都會同步切換到這一檔。若某些面板您已經「鎖定」了特定代碼，那幾個面板不受影響，仍固定顯示各自鎖定的商品。搜尋框本身按 Enter 跳轉、按 **Esc** 或點框外空白處即可關閉（不會執行任何刪單）。

5. **B／S 切換買賣：** 在「非輸入狀態」下（也就是游標沒有停在輸入框、文字框或下拉選單時），直接按 **B** 會把下單面板切成「買進 Buy」，按 **S** 切成「賣出 Sell」。它是透過發送一個下單動作事件（`TICKET_ACTION_EVENT`）通知下單面板切換方向，讓您不必用滑鼠點就能快速調整買賣別。

6. **Esc×2 全部刪單：** 同樣在非輸入狀態下，**在 0.6 秒內連按兩次 Esc**，系統會撤銷所有未成交委託（`cancelAllOrders`），完成後自動刷新交易資料。為避免誤觸，第一次按 Esc 只會跳出提示「再按一次 Esc 全部刪單」，並不會真的刪單；您必須在 0.6 秒內再補一下，才會真正送出全刪。這是盤中要一鍵清掉掛單時很實用的安全設計。

7. **輸入時的快捷鍵行為：** 請留意一個貼心的分工——當您正在任何輸入框（INPUT／TEXTAREA／SELECT）裡打字時，**B、S 與 Esc 全刪單這些單鍵快捷鍵會自動失效**，以免您打字打到「b」就不小心切換買賣、或按 Esc 誤觸刪單；唯獨 ⌘K 例外，它在打字狀態下仍可隨時把您叫進商品搜尋。這樣的設計讓您可以放心在下單欄位輸入數字，又保留隨手叫出搜尋的便利。

### 主題、隱私模式與音效

在 Shioaji Pro 交易終端中，「主題、隱私模式與音效」是把整個介面調校到「您看盤最順眼、最安全、最有臨場感」的一組全域偏好設定。它由三支輕量模組各司其職：主題（theme-store）會把您選的配色與字級以 class 掛到 `<html>` 上，並因為全 App 採 rem 排版，縮放根字級就等於同步縮放整個介面；桌面版還會透過 Tauri 讓原生視窗邊框（標題列外觀）跟著切換。隱私（privacy）模組提供兩個各自獨立的遮蔽開關，透過 `maskAccountId / maskName / maskMoney` 在全 App 一致地把敏感資訊打碼。音效（sounds）模組則用瀏覽器內建的 WebAudio 即時合成成交、委託、警示、錯誤四種提示音，不需要任何音檔。這些設定都各自記在 localStorage，重開程式仍會記得。

以下為您說明如何開啟與操作這個功能：

**如何開啟設定：** 這不是一個可新增的面板，而是常駐在交易終端頂部導覽列（HudHeader）的一顆選單按鈕。請在頂部列點擊「主題」選單（ThemeSettings），即會展開一個下拉，主題模式、漲跌顏色、字級、通知大小、音效與兩個隱私開關都集中在這裡。因為是全域偏好，它會一次套用到整個 App，沒有「連動／鎖定商品」或「彈出獨立視窗」這類面板專屬操作。

**主題模式（深色／純黑／淺色）：** 提供三種底色——深色（dark，預設）、純黑（midnight，更暗、對比更沉，適合暗房或 OLED 長時間盯盤）與淺色（light）。切換後除了介面配色，連下拉選單與捲軸的明暗（colorScheme）都會跟著調整；在桌面版，原生視窗邊框也會同步：深色與純黑對應深色標題列，淺色對應淺色標題列。

**漲跌顏色（紅漲綠跌／綠漲紅跌）：** 這是給習慣不同市場的您準備的。預設是台股慣例「紅漲綠跌」（tw），也可切成國際慣例「綠漲紅跌」（intl）。這個設定不只換文字顏色，連 K 線圖與量能柱（canvas 繪製，需要具體色碼）的漲跌配色都會一起翻轉，讓報價板與圖表口徑一致。

**字級（小／標準／大／特大）：** 提供 0.85 倍（小）、1 倍（標準，預設）、1.15 倍（大）、1.3 倍（特大）四段。由於整個 UI 是以 rem 為單位排版，調字級實際上是縮放根字級（16px × 倍率），因此不只字變大，欄位、間距、整體版面都會等比放大——這對高解析度大螢幕或想拉開資訊密度的您很實用。

**隱私 · 帳號遮蔽：** 選單底部第一個獨立開關。開啟後會把帳號號碼與戶名打碼——帳號會保留末兩碼（例如 `9816502` 顯示成 `•••••02`，這樣多個帳號仍分得出來），姓名則顯示成 `•••`。適合在截圖、螢幕分享或直播盤面時，隱藏身分資訊。此開關預設為關閉。

**隱私 · 金額遮蔽：** 選單底部第二個獨立開關，與帳號遮蔽互不相干、可各自單開。開啟後會把金額類數字（如頂部的銀行水位、損益、權益、數量等）統一遮成 `•••••`。當您只想秀走勢與操作、不想露出實際金額時特別好用。此開關同樣預設為關閉。

**音效開關（成交／委託／警示）：** 一個總開關掌管所有提示音，預設為開啟。開啟時，系統會用 WebAudio 即時合成四種提示：成交是明亮的雙音「叮」（deal）、委託回報是輕柔的一聲點擊（order）、警示是急促連續的高頻嗶聲（alert）、錯誤則是低沉的短促嗡鳴（error）。這些都是程式合成、無需外部音檔；若您在安靜環境或不想被打擾，關掉這個開關即可全部靜音。

**設定即時生效並自動記憶：** 以上每一項調整都是點下去就套用，不需要儲存或重新整理。主題（含模式、漲跌慣例、字級）、帳號遮蔽、金額遮蔽、音效各自寫入獨立的 localStorage 鍵值，因此下次開啟交易終端時，都會沿用您上一回的偏好。

### 通知中心與診斷面板

在 Shioaji Pro 交易終端中，「通知中心 (NoticeCenter)」與「診斷 Debug (DebugPanel)」是一組專為「事後查帳」與「排查連線」而設計的介面系統面板。畫面上跳出的 Toast 提示轉眼就消失，但成交、刪改單、風控攔截、系統訊息這些事件其實都被寫進一份常駐的通知記錄裡——通知中心用 useSyncExternalStore 訂閱這份記錄 (subscribeNoticeLog)，只把這些被寫成通知的事件（成交、刪改單、風控攔截、系統訊息等）依時間排成一條時間軸，讓您回頭核對「剛剛那筆到底成交了沒、系統有沒有報錯」；至於後端推來的「原始委託回報 (order_event)」即時傾印，通知中心並不顯示，那是診斷面板下方「最近 order_event」區塊專屬的內容，也是全 App 唯一會顯示 order_event 的地方。診斷面板則走得更底層：它同時掛上 useStreamStatus 與 SSE 行情流的 tick / order_event 監聽，把 App 版本、連線心跳、行情速率、訂閱數、Token 有效期、伺服器環境等平常看不到的內部狀態攤開來，讓您在「怎麼都不動了」的時候，不必打開開發者工具就能判斷問題出在哪一段。

這兩個面板都是單例 (singleton)、不吃商品的系統面板：它們顯示的是整個 App 與帳戶層級的事件與連線狀態，與您當前選取的商品無關，因此標題列上沒有連動↔鎖定的切換鈕，也不提供彈出獨立視窗的功能。

以下為您說明如何開啟與操作這個功能：

**如何開啟這兩個面板：** 在交易終端頂部導覽列 (HudHeader) 右側，點擊「＋ 新增面板」選單 (AddBlockMenu)。展開後的「新增面板 Add Panel」清單裡，分別選擇「通知中心」與「診斷 Debug」，系統就會把面板加到您目前工作區 (React Grid Layout) 的最底部。由於兩者都是單例面板，工作區裡已經有一個之後，選單中的該項會變灰並標上「（已存在）」，避免重複開啟。加入後可拖曳面板標題列 (PanelChrome 的 drag-handle) 搬移、拉邊調整大小；不需要時點標題列右上角的關閉鈕 (X) 即可移除。

**是否跟隨選取商品：** 這兩個面板都不連動、也不可鎖定。它們不像 K 線圖或五檔會跟著您在自選清單、⌘K 搜尋選到的商品切換，而是永遠呈現全 App 的事件與連線狀態，因此標題列上不會出現連動鈕或鎖定代碼輸入框。

**通知中心｜篩選通知類型：** 面板上方有一排篩選鈕——「全部／成功／錯誤／訊息」。點「成功」只看成交、下單成功這類事件；點「錯誤」快速抓出被拒單、連線失敗、風控攔截等紅字；點「訊息」看一般系統提示。每則通知左側以圖示標色（成功為勾、錯誤為叉、訊息為圓點），中間是 24 小時制的發生時間，右側則是標題與（若有）補充內容，最新的排在最上面。

**通知中心｜清除記錄：** 篩選鈕最右邊有一顆「清除」鈕，按下即清空整份通知記錄；當目前沒有任何通知時，這顆鈕會呈現不可點的停用狀態，清空後面板會顯示「沒有通知」。請注意這是清掉本機的歷史記錄，並不會取消或影響任何實際委託。

**診斷面板｜連線與環境一覽：** 面板上半部是一張狀態表，逐列列出關鍵指標：「App 版本」「方案」（VIP 或 Free）、「GA 即時」（分析追蹤是否啟用）、「SSE 行情流」（LIVE／SYNC／LOST，非 LIVE 時會標成警示色）、「心跳」（距離上次心跳幾秒，超過 15 秒轉警示色代表連線可能卡住）、「行情速率」（過去五秒滑動視窗換算的每秒 Tick 筆數）、「訂閱數」、「API Base」（後端位址，同源時顯示「(同源)」）、「伺服器版本」（後端版本，並標明是「（模擬）」或「（⚠ 正式）」環境）、「Token 有效」（登入憑證還剩幾小時，少於一小時會轉警示色提醒您可能需要重新連線）、以及已載入的「合約數」。這些數值每秒／每隔一段時間自動刷新，凡是異常值都會以警示色標出，方便一眼掃到問題。

**診斷面板｜最近委託回報 (order_event)：** 狀態表下方標題「最近 order_event」，會即時傾印最近幾筆從後端推來的原始委託回報事件，每筆前面附上 24 小時制時間、後面是截斷後的原始 JSON 內容；還沒收到任何事件時顯示「尚無事件」。當您懷疑「明明送出了單，怎麼帳上沒反應」時，這裡能直接看到後端到底有沒有把回報推上來，是排查下單鏈路最直接的一手資料。

**小提醒：** 通知中心與診斷面板都是純粹的「觀測」介面，只讀取並呈現事件與連線狀態，本身不會送出、修改或撤銷任何委託。真正的下單、改單、刪單請仍在下單面板、持倉/委託/帳務等對應面板操作；這兩個面板的價值，在於出狀況時幫您快速定位問題、事後對帳時還原每一步的來龍去脈。

### 首次設定精靈 Onboarding

在 Shioaji Pro 交易終端中，「首次設定精靈 Onboarding」是為了讓全新安裝能一次就正確地連上永豐（Shioaji）而設計的。它是一道「首次啟動關卡」（App.tsx 的 AppGate）：只要本機還沒有存過任何 API 金鑰，程式就不會直接把您丟到一個「看起來空空的、但其實伺服器根本沒被叫起來」的儀表板，而是先擋在這個設定畫面（OnboardingSetup 元件）。您在這裡填入的金鑰只寫進本機 App 資料夾（saveDesktopSettings），接著由它去啟動背後的 Shioaji 伺服器（serverStart sidecar）；啟動後它會持續輪詢伺服器的 /health（reloadWhenHealthy，每 2 秒探一次），一旦健康就自動重新載入頁面、把您帶進真正的交易終端。這是桌面版（Tauri）專屬的流程。

以下為您說明如何開啟與操作這個功能：

**何時會看到它：** 這個畫面不是可新增的面板，也不在「＋ 新增面板」選單裡，您不需要（也無法）手動開啟它。只要是全新安裝、或本機尚未儲存過 API 金鑰，第一次啟動時它就會取代儀表板自動出現；完成設定並成功啟動後就不會再擋您。

**快速匯入 .env（可選）：** 若您手邊已經有寫好金鑰的專案，可點畫面上方的「選資料夾自動讀取 .env」按鈕。請注意它請您選的是「資料夾」而不是 .env 檔本身（因為系統檔案對話框預設會隱藏 .env 這類 dotfile）；選好資料夾後，它會自動在裡面尋找 .env、.env.local 等檔案，並抓出其中的 SJ_API_KEY 與 SJ_SEC_KEY 幫您填好。若資料夾裡有 .env 但找不到這兩把金鑰，畫面會用紅字提示。

**填入 API 金鑰：** 在「API KEY」與「SECRET KEY」兩個欄位分別貼上您的 SJ_API_KEY 與 SJ_SEC_KEY。每個欄位右側都有一顆眼睛按鈕，可切換明碼／遮蔽顯示，方便您核對是否完整貼上、沒有多餘空白或漏字。

**選擇環境（模擬／正式）：** 在「環境」列有「模擬環境」與「正式環境」兩顆按鈕，預設為模擬環境。請注意這裡是您手動選擇要連哪一個環境——一旦切到「正式環境」，畫面會出現紅字提醒「正式環境下單動用真實資金，且需要 Sinopac.pfx 憑證」，代表之後的下單都是真金白銀，請務必確認。

**正式環境的憑證設定：** 只有在您選了「正式環境」時，畫面才會多出「憑證」區塊。請點「選擇 Sinopac.pfx…」挑選您的憑證檔（接受 .pfx／.p12），選好後下方會再出現「憑證密碼」欄位（一樣有眼睛鈕可切換顯示），請填入您下載憑證時設定的密碼。系統在啟動前會做把關：缺金鑰、正式環境卻沒選憑證、或選了憑證卻沒填密碼，都會先擋下並提示原因（例如未填密碼會導致 CA 啟用失敗、下單全部 400），避免您帶著會出錯的設定硬啟動。

**啟動並開始使用：** 都填好後點「啟動並開始使用」。此時畫面會進入啟動中狀態、顯示一條進度動畫並提示「登入與載入合約約需 10–30 秒…」。若金鑰或憑證有問題，它會把伺服器原始日誌翻譯成看得懂的錯誤訊息給您（例如「API Key 不存在或已失效」「金鑰格式錯誤」「憑證載入失敗」「登入失敗—同帳號連線是否已達上限（5 條）」等）；若一切正常，等伺服器回報健康後頁面會自動重新載入，把您帶進交易終端。

**還沒有金鑰？用 AI 助理引導申請：** 畫面右側可能出現一張「AI 助理 — 引導申請 API Key」卡片，並預先幫您填好一句「我是第一次使用，還沒有永豐 Shioaji API Key，可以引導我怎麼申請嗎？」——它只是預填、不會自動送出，您確認後按送出即可讓助理帶您跑申請流程。請注意這個 AI 助理是桌面版專屬的閉源功能模組（透過 FeatureGate 'agent' 載入），若模組未載入（例如開源 Web 版）就不會顯示這張卡片，這時請自行至永豐「API 管理頁」申請金鑰。

**事後還能改：** 這裡填的所有設定日後都不必再走一次精靈。如畫面底部提示所述，金鑰只存在本機、不會上傳；之後若要調整金鑰、環境或憑證，可從交易終端右上角的「伺服器」面板（ServerManager）修改。

### 伺服器管理與環境切換（桌面版）

在 Shioaji Pro 交易終端中，「伺服器管理」是整個終端的引擎室。它不是一張看盤面板，而是常駐在頂部導覽列（HudHeader）上的一顆控制按鈕，負責掌管背後那支跟永豐 Shioaji API 對接的本機伺服器（daemon）——所有報價、下單、帳務資料都是先由這支伺服器登入券商、載入合約後，再串流給前端各面板。它會用 `serverStatus` 每 8 秒、`fetchHealth` 每 15 秒輪詢後端，把運行狀態、PID、連接埠、token 效期與合約載入數即時回報給您。也因為它要在您電腦上啟停一支本機行程，這是「桌面版（Tauri App）」專屬功能；開源 Web 版沒有這支本機伺服器，因此整個伺服器管理按鈕不會出現。

以下為您說明如何開啟與操作這個功能：

**1. 如何開啟「伺服器管理」（桌面版限定）：** 請看交易終端頂部列（HudHeader）。伺服器管理器就常駐在那裡，顯示為一顆帶狀態燈號的按鈕：伺服器正常運行時亮綠燈並標「伺服器」，啟動或連線過程中會轉成旋轉圈圈並標「啟動中…」或「連線中…」，未運行時則是暗燈。點一下這顆按鈕即展開設定浮層（點浮層外的暗色背景或再點一次即可關閉）。此外，從系統匣（menu bar）觸發的相關事件也會自動幫您把這個面板打開。

**2. 讀懂伺服器狀態（健康／PID／連接埠／token／合約）：** 浮層最上方是「Shioaji Server 狀態」，右側標出 Server 版本與 App 版本（若伺服器版本與本 App 要求版本不符，會用琥珀色警告提醒）。狀態列會顯示例如「運行中 · PID 12345 · :8000 · 模擬環境 · 健康」這樣的一行，讓您一眼看清行程編號、連接埠與健康與否；下一行則顯示「token 剩餘 X 小時 · 合約 XX,XXX」，方便您判斷登入權杖還有多久到期、合約表載入了幾筆。

**3. 啟動／重啟／停止伺服器：** 狀態列下方有三顆按鈕——「啟動」「重啟」「停止」。按「啟動」會依您目前設定送出啟動指令，正式環境登入加載合約約需 10–30 秒，成功後系統會在伺服器回報健康時自動重新整理介面，讓每個面板都乾淨地重新連線。「重啟」是先停止、稍候再啟動（這也是您改完環境或憑證後真正套用設定的方式）。「停止」則關閉本機伺服器。動作進行中這三顆按鈕會暫時停用，避免重複點擊。

**4. 設定 API 金鑰與登入資訊：** 「API 金鑰」區有兩個密碼欄位，分別填入您的 `SJ_API_KEY` 與 `SJ_SEC_KEY`（金鑰只存在本機 App 資料夾）。若嫌逐欄貼上麻煩，可按「選資料夾自動讀取 .env」讓它自動抓取。最底下的「登出（清除 API 金鑰）」採兩段式確認——第一次點會變成「再按一次確認登出」，再按一次才真的清除金鑰並重新載入回首次設定畫面，避免誤觸把您登出。

**5. 模擬 ↔ 正式 環境切換：** 「環境」區有「模擬」與「⚠ 正式」兩顆切換鈕。請注意，這是「您要伺服器以哪種環境啟動」的設定，改了之後不會即時生效，必須按「重啟」讓伺服器以新環境重新登入；若目前運行中的環境與您的設定不一致，狀態區也會用紅字提醒您「按『重啟』套用」。切到「正式」時系統會特別以紅字警告「正式環境下單動用真實資金」。另外，頂部列上那顆「模擬環境」（琥珀色徽章）／「正式環境」（醒目的漲色徽章，台式紅漲配色下為紅色）徽章，反映的是伺服器實際登入的環境（由後端 `fetchInfo().simulation` 決定），並非讓您點的開關。

**6. 設定憑證（正式環境下單必要）：** 模擬環境下單不需要憑證，但正式環境一定要。「憑證」區可按「選擇 Sinopac.pfx…」挑選您從券商 API 管理頁下載的憑證檔（選好會顯示檔名並可用 X 清除），下方欄位填入下載憑證時設定的密碼（右側有眼睛圖示可切換顯示／隱藏）。若尚未設定憑證卻選了正式環境，會出現紅字提醒「正式環境無法下單」；若啟動記錄裡偵測到憑證過期或密碼錯誤，也會給出對應的具體提示。

**7. 下單就緒檢查（CA／帳戶）：** 這是專為排查「明明加了憑證卻還是下單失敗」而設計的一鍵自我診斷。按「下單就緒檢查（CA／帳戶）」後，它會列出目前是模擬或正式環境、逐一標示每個證券／期貨帳戶是否已簽署 API 約定書（未簽署者無法下單），並在正式環境下查詢 CA 憑證是否有效與到期日，用 ✓／✗ 清楚呈現。若下單仍失敗，把這段內容連同「複製診斷資訊」一起回報即可。

**8. 開機自動啟動、App 更新與診斷回報：** 浮層下半部還有幾個實用開關：「App 啟動時自動啟動伺服器」可設定每次開 App 自動把伺服器帶起來；「檢查 App 更新」會檢查／下載新版（下載在背景進行，完成後仍由您決定何時按下重新啟動並安裝，不會擅自打斷）；「複製診斷資訊」會把版本、伺服器狀態、串流狀態與最近記錄一次複製到剪貼簿，回報問題時直接貼上即可。

**9. 系統匣面板（menu bar，桌面版隨手看盤）：** 桌面版另附一個常駐系統匣的精簡下拉面板（TrayPanel），讓您不必打開主視窗也能瞄一眼行情。它頂端顯示「Shioaji Pro」與未實現損益總額，右側齒輪鈕可自訂要顯示哪幾區（持倉損益／自選清單／排行榜）以及是否顯示小線圖，另一顆放大鈕可開啟主視窗。面板內的持倉、自選、漲幅排行都可捲動瀏覽；重點是——在系統匣裡點任何一檔商品，都會把主視窗叫到前景並將該商品連動到終端各面板（等同幫您在主視窗選定了商品）。按 Esc 可收起這個小面板。

### AI Agent 與策略回測

在 Shioaji Pro 交易終端中，「AI Agent」與「策略回測」是兩個進階分析模組，專為需要把「想法自動化」的使用者而設計：AI Agent 讓您用對話驅動一位能操作、能觀察、能被排程的交易助理；策略回測則讓您把自己寫的策略拿當日與歷史資料跑一遍，先看清成本與績效，再決定要不要真的下場。這兩者在技術上都是**桌面版專屬的閉源模組**——它們不放在公開的原始碼裡，而是透過私有的 `@modules` 清單動態載入（`agentModule.Panel` 與 `backtestModule.Panel`），並由 `FeatureGate` / `useFeature()` 的分級授權機制（`features.ts`）決定您這台裝置、這個帳號能不能開。也因為如此，開源的 Web 版本身不含這兩個模組，開啟時只會看到鎖定畫面而非功能本體。

以下為您說明如何開啟與操作這個功能：

**新增面板：** 在交易終端頂部導覽列（HudHeader）右側，點擊「＋ 新增面板」選單（AddBlockMenu）。在彈出的「新增面板 Add Panel」清單中，選擇 「AI Agent」 或 「策略回測」，系統即會在您目前的工作區（React Grid Layout）底部新增對應面板。這兩個面板都是**單例（singleton）**，工作區裡各自只能存在一個；若已經開過，選單中該項會呈現灰色 disabled 並標註「（已存在）」，無法重複新增。

**桌面版限定（很重要）：** 如果您在開源 Web 版開啟這兩個面板，由於閉源模組不在該環境中，`FeatureGate` 會渲染一張鎖定畫面——顯示「下載桌面版」圖示、標題寫著「AI Agent 為桌面版專屬功能」或「策略回測 為桌面版專屬功能」，下方附上該功能的官方說明，以及一個「下載桌面版 →」的連結（指向 GitHub Releases）。也就是說，這兩項功能請在桌面版使用，Web 版並不提供其實際運作。

**權限門檻（VIP）：** 即使在桌面版，這兩項也屬於 VIP 分級功能。授權由關閉式解析器依您帳號的 `person_id` 判定會員等級（`free` / `vip`），結果會快取在瀏覽器 `sj-tier-cache` 以避免每次開啟時閃一下鎖定畫面，未取得前預設為 `free`。若您的等級尚未達標，`FeatureGate` 會改以鎖頭圖示顯示「AI Agent／策略回測 為 VIP 專屬功能」，並提示「請聯繫您的營業員了解 VIP 方案」。本功能非免費、也不保證所有帳號皆可使用，實際開通以您的方案與營業員說明為準。

**AI Agent 面板做什麼：** 依官方說明，AI Agent 提供**多供應商的 agentic 對話**（模組支援 Anthropic、OpenAI、Codex 等不同 AI 供應商，可設定預設供應商）、一個**技能市集**、**排程任務**（模組內建 `ensureScheduler`，可讓助理按時執行），以及**從您的操作中觀察學習**。由於這是閉源模組，面板內部的實際分頁與細部操作以您桌面版當下載入的版本為準，這裡不臆測其中的個別按鈕。

**策略回測面板做什麼：** 依官方說明，策略回測讓您**自行撰寫策略（JS ＋ ta 技術指標函式庫）**，交由一具**內含交易成本的回測引擎**運算，並提供**多商品的整合績效**檢視。面板會帶入一個商品合約作為回測標的，並可把您在面板中選定的商品透過 `onPick` 連動回整個終端（例如同步切換 K 線圖與下單面板的商品）。

**面板特性（不連動、不可彈出）：** 這兩個面板都不是 pinnable 面板，標題列上**沒有「連動↔鎖定」的切換鈕**——因此您無法把面板凍結在某一檔商品上。不過要留意，兩者對「全域選取商品」的行為並不一樣：**AI Agent 面板不接收商品合約**（載入時只是 `<Panel/>`，沒有 contract），確實不像 K 線圖那樣自動跟隨全域選取，純以對話為主；**策略回測面板則相反**——它的回測標的預設會**帶入並跟隨您全域選取的商品**（`useBlockContract` 在面板 pin 為 null 時回傳全域選取的 `selected`，再以 `<BtPanel contract={contract}/>` 傳入，並隨全域選取變動而更新），您可以再於面板內另選其他標的。兩者也都不在可彈出清單（POPOUT_TYPES）內，因此標題列**沒有「彈出獨立視窗」按鈕**，無法拉到第二螢幕；標題列僅提供拖曳搬移與右上角的「✕ 移除此面板」。要重新開啟時，回到「＋ 新增面板」選單再點一次即可。

## 安全與客戶端限制（務必了解）

- **停損／停利／到價警示／括號單 OCO** 都是「**客戶端觸價**」：由 App 在您電腦上監看即時行情達成，**只在程式開啟且行情連線正常時才會運作，關掉程式或斷線即失效**。需長時間離場，切勿把重要停損完全託付給它們。
- **鋪單「動態追價」、組合單「到價監控」** 同屬客戶端定時迴圈，關掉程式即停止自動撤補／送單。
- **閃電下單、鋪單** 送出的是真實委託，皆有安全鎖（預設鎖定、斷線自動上鎖、Esc 隨手鎖）。
- **風控 Kill Switch** 攔的是「新單送出」；**Esc×2** 清的是「已掛在市場上的未成交委託」，兩者用途不同、可搭配。
- **預設為模擬環境**（假錢）；切正式環境需 CA 憑證，之後每一筆都是真實交易，風險自負。
- **AI Agent 與策略回測** 為桌面版專屬（閉源）模組，開源 Web 版顯示鎖定畫面。
